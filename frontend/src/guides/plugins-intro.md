# 插件介绍

## TP 传送指令

TPPlus 重写了 TShock 的传送功能，支持传送模式管理。

### 基础指令

| 指令 | 说明 |
|------|------|
| `/tp <玩家名>` | 向目标玩家发起传送请求或直接传送 |
| `/tpa` | **同意**当前待处理的传送请求 |
| `/tpd` | **拒绝**当前待处理的传送请求 |

### 传送模式

通过 `/tpmode <模式>` 设置别人传送你时需要的行为：

| 模式 | 效果 |
|------|------|
| `agree` (允许) | 别人可直接 `/tp` 到你位置，无需确认 |
| `request` (需同意) | 别人 `/tp` 你需要你输入 `/tpa` 同意 |
| `block` (拒绝) | 拒绝所有人的传送请求 |

管理员可用 `/tpmode setdef <a\|r\|b>` 设置全局默认模式。

## 领地插件 (House)

House 插件用于保护你的建筑不被他人破坏。

### 创建领地

1. 选取两个对角点：
   - `/house set 1` — 敲击一个方块标记左上角
   - `/house set 2` — 敲击一个方块标记右下角
2. 输入 `/house add <领地名称>` 创建

> 创建后请记下领地名称，后续所有操作都需要它。

### 管理指令

| 指令 | 说明 |
|------|------|
| `/house delete <名称>` | 删除自己的领地 |
| `/house redefine <名称>` | 重新选取两个角点后，重新定义区域 |
| `/house list [页码]` | 列出自己拥有的领地 |
| `/house info <名称>` | 查看领地详细信息（作者、拥有者、使用者、锁状态） |
| `/house clear` | 清除已选取的敲击点 |

### 分享权限

| 指令 | 说明 |
|------|------|
| `/house allow <玩家名> <名称>` | 添加**拥有者**（可管理领地） |
| `/house disallow <玩家名> <名称>` | 移除拥有者 |
| `/house adduser <玩家名> <名称>` | 添加**使用者**（仅可在领地内建造） |
| `/house deluser <玩家名> <名称>` | 移除使用者 |

### 其他功能

| 指令 | 说明 |
|------|------|
| `/house lock <名称>` | 开关锁（上锁后只有拥有者/使用者可使用箱子） |
| `/house show <名称>` | 显示指定领地的边界框 |
| `/house showall` | 显示所有领地的边界框 |
| `/house auto` | 进入领地时自动显示边界 |
| `/house tp <名称>` | 传送到指定领地 |
| `/house name` | 敲击一个方块查看它属于哪个领地 |

### 权限级别

| 级别 | 可执行操作 |
|------|-----------|
| **作者** | 创建者，拥有全部权限（删除、修改、分享、锁） |
| **拥有者** | 通过 `/house allow` 添加，可管理领地、修改使用者 |
| **使用者** | 通过 `/house adduser` 添加，仅可在领地内建造/使用设施 |
| **其他玩家** | 不可破坏、不可放置（领地保护范围内） |
