# 玩家在线时长统计系统 — 实施计划（修订版）

## 1. 目标
在 TsWeb 项目中实现完整的玩家在线时长统计系统。插件端负责数据采集 + REST API 暴露；后端通过 REST API 代理查询；前端展示排行榜、逐时在线、日历热力图。

---

## 2. 架构关键变更

| 原方案 | 修订方案 | 原因 |
|--------|----------|------|
| 后端 better-sqlite3 直连 tshock.sqlite | 后端通过插件 REST API 查询 | 后端在远端，无法直接访问数据库文件 |
| `hourly_online_snapshot.online_players` 存用户名 | 改为存玩家数字 ID（空格分隔） | 用户名可能包含危险字符 |
| — | 删除 `check_db.js` 和 `view_users.js` | 清除旧数据库路径逻辑 |

## 4. 实施步骤

### Phase 1：插件端 — 数据采集核心（重写 `plugin/OnlineData.cs`）

**类设计**：
- 命名空间 `TShockData`，静态类 `OnlineData`
- 参考 `PlayerStats.cs` 的 REST handler 模式（`SecureRestCommand` + `RestRequestArgs` + `RestObject`）

**4.1.1 建表逻辑（`Initialize()`）**

执行两条 `CREATE TABLE IF NOT EXISTS`：

```sql
CREATE TABLE IF NOT EXISTS player_daily_stat (
    uid TEXT NOT NULL,
    date TEXT NOT NULL,
    daily_min INTEGER NOT NULL DEFAULT 0,
    PRIMARY KEY (uid, date)
);
CREATE INDEX IF NOT EXISTS idx_pds_uid ON player_daily_stat(uid);

CREATE TABLE IF NOT EXISTS hourly_online_snapshot (
    hour_ts INTEGER PRIMARY KEY,
    online_ids TEXT NOT NULL DEFAULT ''
);
CREATE INDEX IF NOT EXISTS idx_hour_ts ON hourly_online_snapshot(hour_ts);
```

`online_ids` 改为存玩家数字 ID（从 `TShock.UserAccounts.GetUserAccountByName(name)?.ID` 获取），空格分隔。

**4.1.2 定时任务**

使用 `ServerApi.Hooks.GameUpdate` + `long _tickCounter` 帧计数：

- **每分钟触发**（`_tickCounter % 3600 == 0`，60fps×60s）：
  - 遍历 `TShock.Players`，过滤 `player != null && player.Active && player.IsLoggedIn`
  - 获取当前日期 `yyyy-MM-dd`（`DateTime.Now.ToString("yyyy-MM-dd")`）
  - 对每个已登录玩家执行：
    ```sql
    INSERT INTO player_daily_stat (uid, date, daily_min) VALUES (@0, @1, 1)
    ON CONFLICT(uid, date) DO UPDATE SET daily_min = daily_min + 1
    ```

- **每小时整点触发**（`_tickCounter % 216000 == 0`，60fps×3600s）：
  - 计算 `hour_ts = now.Year*1000000 + now.Month*10000 + now.Day*100 + now.Hour`
  - 收集在线已登录玩家的数字 ID，用空格拼接
  - 获取 ID 方式：`TShock.UserAccounts.GetUserAccountByName(player.Name)?.ID`
  - 执行：
    ```sql
    INSERT INTO hourly_online_snapshot (hour_ts, online_ids) VALUES (@0, @1)
    ON CONFLICT(hour_ts) DO UPDATE SET online_ids = @1
    ```

**4.1.3 玩家加入 Hook（`ServerApi.Hooks.ServerJoin`）**
- 当玩家加入时，将其数字 ID 追加到当前小时记录的 `online_ids` 中
- 逻辑：读取当前小时的已有 `online_ids` → 检查 ID 是否已存在 → 拼接 → UPDATE
- 使用 `TShock.DB.QueryReader()` 读取，`TShock.DB.Query()` 更新

**4.1.4 三个 REST API 端点**

参考 `PlayerStats.cs` 模式：

1. **`/data/online/hourly?date=2026-06-26`**
   - 参数校验：解析日期，生成 `startTs = yyyyMMdd00`、`endTs = yyyyMMdd23`
   - SQL：
     ```sql
     SELECT hour_ts, online_ids FROM hourly_online_snapshot 
     WHERE hour_ts >= @0 AND hour_ts <= @1 ORDER BY hour_ts
     ```
   - 使用 `QueryReader` 逐行读取，构建 `List<Dict>` 返回
   - 返回 `RestObject`：`{ "hours": [ { "hour_ts": 2026062614, "online_ids": [1, 5, 12] }, ... ] }`
   - **注意**：`online_ids` 空格分隔字符串需在插件端 `Split(' ')` 转为 int 数组再返回

2. **`/data/online/ranking?days=30`**
   - 计算 `since = DateTime.Now.AddDays(-days).ToString("yyyy-MM-dd")`
   - SQL：
     ```sql
     SELECT uid, SUM(daily_min) AS total_min FROM player_daily_stat 
     WHERE date >= @0 GROUP BY uid ORDER BY total_min DESC LIMIT 100
     ```
   - 返回 `{ "ranking": [ { "uid": "player1", "total_min": 360 }, ... ] }`

3. **`/data/online/player?name=xxx&year=2026`**
   - 参数校验 year
   - SQL：
     ```sql
     SELECT date, daily_min FROM player_daily_stat 
     WHERE uid = @0 AND date >= @1 AND date <= @2 ORDER BY date
     ```
   - `@1` = `year-01-01`，`@2` = `year-12-31`
   - 返回 `{ "player": "xxx", "year": 2026, "days": [ { "date": "2026-06-26", "daily_min": 45 }, ... ], "total_min": 360 }`
   - **注意**：即使某天没有记录也无需补 0，前端自行处理缺失日期为 0。

**权限**：三个端点使用 `"data.rest.invsee"` 权限（与现有统计端点一致）。

**4.1.5 `Dispose()` 方法**
- `ServerApi.Hooks.GameUpdate.Deregister(this, OnGameUpdate)`
- `ServerApi.Hooks.ServerJoin.Deregister(this, OnServerJoin)`

### Phase 3：后端 — Service 层（新建 `backend/services/onlineService.js`）

**模式**：完全参照 `tshockService.js` 的 HTTP 调用模式，通过 TShock REST API 获取数据。

```js
import { getConfig, onConfigUpdate } from '../config.js'

let baseUrl = 'http://localhost:7878'
let apiKey = ''

export class OnlineService {
  constructor() {
    onConfigUpdate((config) => {
      const host = config.tshock?.host || 'localhost'
      const h = host.startsWith('http') ? host : `http://${host}`
      baseUrl = `${h}:${config.tshock?.port || 7878}`
      apiKey = config.tshock?.apiKey || ''
    })
  }

  async getHourlyOnline(date) { ... }
  async getRanking(days) { ... }
  async getPlayerCalendar(name, year) { ... }
}
```

每个方法构造 URL 格式：
```
{baseUrl}/data/online/hourly?date={date}&token={apiKey}
```
使用 `fetch()` GET 请求，返回 JSON。

**无 better-sqlite3 依赖，无本地文件访问。**

│ │排名│玩家  │时长 │ │  ▐▐  ▐▐▐  ▐▐▐▐          │
│ │ 1  │xxx   │6h  │ │  悬停显示玩家ID列表       │
│ │ 2  │yyy   │4h  │ │                          │

├────────────────────┴─────────────────────────┤
│  玩家日历热力图                                │
│  玩家: [______]  年份: [2026 ▼]               │
│  ┌──────────────────────────────────────┐     │
│  │ 日  □□■□□■■□■□□ ... (53列×7行)       │     │
│  │ 一  □□□■□□□■□□ ...                    │     │
│  │ ...                                    │     │

**区域 1 — 排行榜**：
- `<select>` 天数选择（7/14/30/90），默认 30
- `<table>` 显示排名、玩家名、总时长
- 时长格式化：`Math.floor(min/60) + 'h ' + (min%60) + 'm'`
- 加载状态：`v-if="rankingLoading"` 显示加载提示

**区域 2 — 逐时在线**：
- `<input type="date">` 日期选择，默认今天
- 24 根柱子的柱状图，纯 CSS（`div` + `style="height: {count*4}px"`）
- 每个柱子 `.bar` 绝对定位，底部对齐
- 鼠标悬停 `@mouseenter` 显示 tooltip（玩家 ID 列表）
- 柱子颜色：渐变绿色

**区域 3 — 日历热力图**：
- 玩家名 `<input>` + 年份 `<select>`
- 7 行 × 最多 53 列 CSS Grid
- 每格按 `daily_min` 分 5 档着色：
  - 0：`#ebedf0`（无数据灰）
  - 1-30：`#9be9a8`（浅绿）
  - 31-60：`#40c463`（中绿）
  - 61-120：`#30a14e`（深绿）
  - 120+：`#216e39`（最深绿）
- Grid 布局：每列为一个周（周日到周六），需要计算该年第一天是周几来确定偏移
- `title` 属性或自定义 tooltip 显示日期和分钟数
- 底部色阶图例

**空状态处理**：无数据时显示"暂无数据"提示，不报错。

**API 调用**：
```js
import { get } from '../../utils/api.js'
// 排行榜
const rankingRes = await get(`/api/online/ranking?days=${days.value}`)
// 逐时在线
const hourlyRes = await get(`/api/online/hourly?date=${date.value}`)
// 玩家日历
const calendarRes = await get(`/api/online/player?name=${name.value}&year=${year.value}`)
```
