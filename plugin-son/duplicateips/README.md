# DuplicateIPs 查小号

TShock 服务器插件，用于查询玩家账号的关联关系（基于IP和UUID）。



## 可用命令

| 命令 | 功能描述 | 示例 |
|------|----------|------|
| `/dip <玩家名>` | 查询指定玩家的关联账号（基于IP和UUID） | `/dip Player1` |
| `/dip *` | 查询所有共享IP/UUID的关联账号组 | `/dip *` |
| `/dip` | 显示帮助信息 | `/dip` |



## 关联算法

插件使用并查集（Union-Find）数据结构进行高效的关联分析：

1. **数据收集**：从数据库读取所有玩家的 ID、用户名、UUID 和 KnownIPs 信息
2. **IP关联**：共享相同IP地址的账号建立关联关系
3. **UUID关联**：共享相同UUID的账号建立关联关系
4. **合并操作**：通过并查集的 Union 操作将有共同关联的账号合并到同一组
5. **查询操作**：通过 Find 操作快速定位目标玩家所在的关联组

### 关联判定规则

| 关联类型 | 判定条件 | 说明 |
|----------|----------|------|
| IP关联 | 两个账号的 KnownIPs 列表有交集 | 同一IP地址登录的多个账号 |
| UUID关联 | 两个账号的 UUID 相同 | 同一设备/客户端创建的多个账号 |

### 传递性特性

关联关系具有传递性：
- 若 A 和 B 共享IP，则 A ↔ B
- 若 B 和 C 共享UUID，则 B ↔ C
- 因此 A、B、C 都属于同一关联组

### 时间复杂度

- 预处理（Union操作）：近似 O(α(n))，α 为阿克曼函数的反函数，接近常数
- 查询（Find操作）：近似 O(α(n))
- 总体复杂度：O(n × α(n))，n 为玩家数量

## 配置说明

### 默认配置

插件无需额外配置文件，安装后即可使用，默认行为如下：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| IP关联 | 启用 | 基于KnownIPs进行账号关联 |
| UUID关联 | 启用 | 基于UUID进行账号关联 |
| 颜色输出 | 启用 | 使用彩虹颜色区分不同关联组 |
| 权限节点 | `tshock.admin.ban` | 执行命令所需的权限 |

### 颜色配置

插件内置7种明亮易读的彩虹颜色，自动循环使用：

| 序号 | 颜色代码 | 颜色名称 |
|------|----------|----------|
| 1 | FF6B6B | 珊瑚红 |
| 2 | FFA07A | 浅三文鱼色 |
| 3 | FFD700 | 金色 |
| 4 | 98FB98 | 苍白绿 |
| 5 | 87CEEB | 天蓝色 |
| 6 | 9370DB | 梅紫色 |
| 7 | DDA0DD | 梅红色 |

### 输出格式

**查询单个玩家:**
```
=== 玩家 Player1 的关联分析 ===
关联账号 (3 个):
  - Player1 (目标)
  - Player2
  - Player3
共享IP (2 个): 192.168.1.100, 10.0.0.5
```

**查询所有关联组:**
```
=== 关联组 #1 ===
账号: Player1, Player2, Player3
共享IP: 192.168.1.100
=== 关联组 #2 ===
账号: Admin, TestUser
共享IP: 10.0.0.1, 10.0.0.2
```
