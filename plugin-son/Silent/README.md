# Silent 插件

管理员无痕进服插件，支持隐身、无敌、隐藏进出消息。可以配合占卜球实现无痕监控

## 功能特性

- **命令响应** - 无法被大多数命令响应，包括服务端的/who
- **隐身** - 其他玩家无法看到静默玩家及其创建的所有实体（弹幕，被杀信息）
- **无敌** - 静默玩家受到保护
- **隐藏进服退服消息** - 其他玩家看不到静默玩家进入和退出服务器
- **离线配置** - 支持为离线玩家预配置静默模式，下次进服自动生效

## 缺陷
- **命令响应** - 大部分插件对Main.player[x].active的没有做全程适配，请尽量避免对静默对象使用命令或者让静默对象执行指令（例如/g非常不稳定，物品大概率掉地上且静默对象不可拾取，建议直接使用修改器获取物品）
- **聊天** - 为防止误发，直接禁用了静默对象的聊天功能
- **隐蔽性** - 静默对象进服时会有短暂的时间被看到，超过100ms，因此会被其他玩家看到
- **杂项** - 静默对象无法被传送机传送；静默对象无法拾取物品，但是丢出的物品会被其他玩家拾取；静默对象吃生命水晶吃药的浮动数字会显示；必定导致怪物索敌错误

## 用法

| 命令 | 别名 | 说明 | 权限 |
|------|------|------|------|
| `/silent add <玩家名>` | `/st add` | 添加玩家到无痕列表 | `silent.use` |
| `/silent del <玩家名>` | `/st del` | 从无痕列表移除玩家 | `silent.use` |
| `/silent list` | `/st list` | 显示无痕列表 | `silent.use` |

## 技术实现

- 使用 `Main.player[playerIndex].active = false` 实现玩家隐身
- 通过 `GodMode` 属性实现无敌
- 使用 `SilentJoinInProgress` 和 `SilentKickInProgress` 隐藏进出消息
- Hook `NetMessage.SendData` 拦截 `PlayerActive` 数据包防止状态被覆盖

## 安装

将编译好的 `Silent.dll` 放入 `ServerPlugins` 目录。

## 权限

需要 `silent.use` 权限才能使用命令。默认情况下管理员拥有此权限。
