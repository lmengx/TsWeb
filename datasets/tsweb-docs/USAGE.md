# TSWeb 文档使用说明

> 本文档指导你如何将 TSWeb 知识库导入 Dify 平台，并提供提示词模板和使用场景建议。

---

## 一、Dify 导入步骤

### 前置条件

- 拥有 Dify 平台访问权限（本地部署版或云端版均可）
- 已获取 `tsweb-docs/` 目录下的所有 Markdown 文件

### 步骤详解

#### Step 1: 进入知识库

登录 Dify → 左侧导航栏 → **知识库** → **创建知识库**

#### Step 2: 上传文档

选择 **导入现有文档** → 上传方式可选：

- **拖拽上传**：将本目录的 `.md` 文件拖入上传区域
- **选择文件**：点击上传区域，选择文件（可多选：按住 Ctrl/Cmd 点击多个文件）
- **ZIP 批量**：将整个 `tsweb-docs/` 目录打包为 ZIP，直接上传 ZIP

> **建议**：将 `01_项目概述.md` 和 `02_插件架构与模块.md` 一起导入，以保证上下文完整。

#### Step 3: 索引配置

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| 索引方式 | **高质量** | Embedding 模型处理，检索精度最高 |
| 分段方式 | **自定义** | 手动控制分段策略 |
| 分段规则 | 按 `#` 标题分段 | 每个一级/二级标题作为一个独立段落 |
| 分段最大长度 | **1000 tokens** | 针对代码和 API 路由表，不宜过长 |
| 分段重叠长度 | **100 tokens** | 保证段落间上下文衔接 |
| 检索策略 | **混合检索** | 关键字精确匹配 + 语义向量检索 |

#### Step 4: 确认导入

点击 **开始处理** → 等待索引完成 → 在知识库中测试检索效果

## 二、提示词模板

### 模板 1：项目概览查询

```
你是一个 TShock 插件专家。请根据提供的知识库内容回答以下问题：

问题：{用户问题}

请从以下方面回答：
1. 如果涉及架构，请说明 TSWeb 的四层架构
2. 如果涉及功能，请列出相关模块
3. 如果涉及 API，请提供具体的路由路径
4. 引用知识库中的具体内容作为依据
```

### 模板 2：API 路由查询

```
请根据 TSWeb 知识库，列出与 {功能关键词} 相关的所有 REST API 路由。

要求：
- 给出完整的路由路径（如 /data/users/*）
- 说明每个路由的用途
- 说明所需的权限（如有）
- 按功能分类组织输出
```

### 模板 3：模块功能咨询

```
请详细说明 TSWeb 中 {模块名称} 模块的功能和实现方式。

要求：
- 该模块在 Main.cs 中的注册位置
- 提供的 API 接口或聊天命令
- 相关的配置项（如有）
- 在插件架构中的层级关系
```

### 模板 4：架构分析

```
请分析 TSWeb 的 {四层架构 / 模块注册流程 / 路由体系 / 生命周期}。

要求：
- 详细描述架构设计
- 列出关键组件或模块
- 说明组件间的交互关系
- 给出代码层面的依据
```

## 三、使用场景

### 场景 1：新人上手 - 了解项目全貌

**适用人员**：新加入项目的开发者、需要了解 TSWeb 功能的服主

**推荐问题**：
- "TSWeb 是什么？有哪些核心功能？"
- "TSWeb 的四层架构分别是什么？各自的作用？"
- "TSWeb 使用了哪些技术栈？"
- "列出 TSWeb 的所有功能模块"

### 场景 2：API 调试 - 查询接口信息

**适用人员**：需要调用 TSWeb REST API 进行二次开发或调试

**推荐问题**：
- "列出所有 /data/users/* 相关的 API 路由"
- "反作弊模块有哪些 API 接口？"
- "文件管理模块提供了哪些操作？"
- "在线统计功能的数据接口有哪些？"

### 场景 3：故障排查 - 理解模块行为

**适用人员**：遇到插件异常、需要理解模块实现逻辑的开发者

**推荐问题**：
- "Main.cs 的 Initialize() 方法中模块的注册顺序是怎样的？"
- "OnReload 重载事件会重新加载哪些配置？"
- "Dispose 方法中做了哪些清理操作？"
- "TSWeb 如何清理注册的聊天命令和 REST 路由？"

### 场景 4：功能扩展 - 参考架构设计

**适用人员**：计划为 TSWeb 贡献代码或开发子插件的开发者

**推荐问题**：
- "TSWeb 的聊天命令有哪些？它们各自的权限是什么？"
- "如何注册一个新的 REST API 路由？参考现有路由的写法"
- "plugin-son 子插件的架构是怎样的？"
- "消息格式规范是什么？"

### 场景 5：AI 辅助开发 - 代码生成参考

**适用人员**：使用 AI 工具辅助 TSWeb 开发的开发者

**推荐提示词**：
```
请参考 TSWeb 项目中 GroupOP 模块的实现方式（REST API 注册 + CRUD），
帮我实现一个新的 {模块名称} 模块。需要包含：
1. 完整的 C# 类代码
2. REST API 路由注册
3. 错误处理
4. 参考 TSWeb 的代码风格
```

## 四、常见问题

### Q: 导入后检索不到具体路由信息？

**A**: 检查分段设置。`02_插件架构与模块.md` 中的路由表较长，建议：
- 分段最大长度设为 1000-2000 tokens
- 分段重叠设为 100-200 tokens
- 检索策略使用"混合检索"

### Q: 如何提高检索精度？

**A**: 
- 使用**高质量**索引模式
- 设置合理的分段规则（按 `#` 标题分段）
- 在提示词中明确指定要查询的知识领域
- 使用模板中的提示词格式

### Q: 是否支持图片/图表检索？

**A**: 当前文档以 Markdown 文本为主，不含图片。Dify 对纯文本 Markdown 的支持效果最好。
