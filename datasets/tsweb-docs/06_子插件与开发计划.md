# 子插件与开发计划

## plugin-son 子插件

位于 `plugin-son/` 目录，作为 TSWeb 的独立子插件集，每个子插件独立编译、独立加载。

---

| 状态 | 已创建 |

#### 5. House — 领地

**路径**：`plugin-son/House/`

| 项目 | 内容 |
|------|------|
| 文件 | `Plugin.cs`, `Config.cs`, `Customize.cs`, `PacketReceive.cs`, `Utils.cs`, `HouseRegion.csproj` |
| 国际化 | `i18n/` — en-US, es-ES, ru-RU 多语言支持 |
| 状态 | 已编译 Debug + Release |

**当前功能**：
- `/house set 1/2` 敲击选点创建领地
- `/house add/delete/redefine` 管理领地
- `/house allow/disallow/adduser/deluser` 权限分享（拥有者/使用者）
- `/house lock/show/showall/auto/name/tp` 领地管理
- 边界粒子显示、自动显示
- 多权限级别（作者/拥有者/使用者）
- 箱子锁功能

**待重写**（详见开发计划）：

| 改进项 | 说明 |
|--------|------|
| 选点方式 | 改为 `/h c 创建`，玩家自身坐标控制粒子框选，或敲击选定第二个点 |
| 指令精简 | `/h c` 创建, `/h tp` 传送, `/h del` 删除, `/h set` 设置 |
| 子领地 | 新圈领地完全位于旧领地中时自动设为子领地，独立权限 |
| 传送匹配 | 精确匹配 → 自身同名 → 公开同名 → 自身模糊 → 公开模糊 |
| 公开地标 | `/h 地标 名称` 注册全局唯一公开名称 |
| 转让 | `/h transfer 领地名 目标玩家` |
| 传送点 | `/h settp` 设置当前位置为传送点 |
| 驱离点 | `/h set 字段 值` 配置各权限项 |

#### 6. MapManager — 地图管理

**路径**：`plugin-son/MapManager/`

#### 9. duplicateips — 重复 IP

**路径**：`plugin-son/duplicateips/`

| 项目 | 内容 |
|------|------|
| 文件 | `Main.cs`, `duplicateips.csproj`, `duplicateips.slnx`, `README.md` |
| 状态 | 已编译 Release |
| 功能 | 检测重复 IP 玩家，查询相同 IP 下的所有账号 |

**相关 REST API**：
| 端点 | 方法 | 说明 |
|------|------|------|

| `/data/users/allduplicateips` | GET | 查询所有重复 IP 记录 |

#### 10. tp-plus — 传送增强

**路径**：`plugin-son/tp-plus/`

| 项目 | 内容 |
|------|------|
| 文件 | `TeleportRequest.cs`, `Config.cs`, `TPModeStore.cs`, `TPRequest.cs` |
| 状态 | 已创建 |

**功能**：
- 重写 TShock 传送功能
- 传送模式管理：`/tpmode agree`（允许）/ `request`（需同意）/ `block`（拒绝）
- 管理员可设置全局默认模式：`/tpmode setdef <a|r|b>`
- 传送请求确认机制
