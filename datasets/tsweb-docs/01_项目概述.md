# TSWeb 项目概述

---

## 项目定位

**TSWeb**（全称 TShockData）是一个 **TShock REST Web 管理插件**，基于 TerrariaPlugin 框架开发。它为 Terraria 服务器提供了一站式的 **Web 管理面板**，通过浏览器即可完成服务器配置、玩家管理、反作弊监控、数据分析等操作。

| 属性 | 值 |
|------|-----|
| 项目名称 | TSWeb (TShockData) |
| 类型 | TShock REST Web 管理插件 (TerrariaPlugin) |
| 作者 | lmx12330 |
| 版本 | 1.0.0.0 |
| TShock API 版本 | 2.1 |
| 描述 | TShockRESTweb管理 |
| 项目主页 | https://github.com/lyt/TsWeb |

## 四层架构

TSWeb 采用前后端分离 + 插件扩展的四层架构设计：

```
┌─────────────────────────────────────────────────────┐
│                    plugin/ (C#)                       │
│           TShock 插件核心 (TerrariaPlugin)              │
│   负责: REST API 服务、聊天命令、游戏事件钩子、反作弊     │
├─────────────────────────────────────────────────────┤
│                  frontend/ (Vue 3)                    │
│            Web 管理面板 (用户界面)                       │
│   负责: 管理界面展示、用户交互、数据可视化                 │
├─────────────────────────────────────────────────────┤
│                  backend/ (Node.js)                    │
│            Express 中间层 (API 代理与增强)               │
│   负责: TShock REST API 代理、认证鉴权、文件服务、     │
│          数据库管理、初始化配置                          │
├─────────────────────────────────────────────────────┤
│                plugin-son/ (子插件集)                    │
│            子功能插件集合 (可插拔扩展)                    │
│   包括: AntiProxy, AutoRegister, ChestFix, HotReload,  │
│          House, MapManager, PlanOff, Silent,            │
│          duplicateips, koishi, tp-plus 等               │
└─────────────────────────────────────────────────────┘
```

### 各层职能

#### 1. plugin/ — C# TShock 插件核心

- **语言/框架**: C# (.NET), TShockAPI
- **入口**: `Main.cs` → `TShockData` 类继承 `TerrariaPlugin`
- **职责**:
  - 注册 REST API 路由（40+ 条）
  - 注册游戏内聊天命令
  - 挂载游戏事件钩子（弹幕、物品、玩家行为等）
  - 反作弊检测与限制
  - BOSS 限制、自动注册、QQ 绑定等核心功能
  - 数据库交互（通过 TShockAPI 操作 SQLite）

#### 2. frontend/ — Vue 3 Web 管理面板

- **语言/框架**: Vue 3 + Vite + vue-router + marked
- **构建工具**: Vite 6
- **包管理器**: pnpm
- **职责**:
  - Web 管理界面展示
  - 路由管理（登录、控制台、设置等子页面）
  - API 调用与数据渲染
  - Terminal 控制台界面
  - 文件管理、物品搜索、反作弊配置等管理功能

#### 3. backend/ — Node.js Express 中间层

- **语言/框架**: Node.js, Express 5
- **包管理器**: pnpm
- **主要依赖**: better-sqlite3, sql.js, jsonwebtoken, bcrypt, iconv-lite, node-forge, cors
- **职责**:
  - TShock REST API 代理转发（解决跨域和认证问题）
  - 用户认证（JWT Token）
  - 初始化配置向导（首次启动 Setup 流程）
  - 文件访问白名单管理
  - IP 地理位置查询代理
  - 静态文件服务（托管前端构建产物）

#### 4. plugin-son/ — 子插件集

- **语言**: C# (.NET TShock Plugin)
- **类型**: 可选子插件集合，按需加载
- **包含插件**:
  - **AntiProxy**: 反代理检测
  - **AutoRegister**: 自动注册
  - **ChestFix**: 箱子修复
  - **HotReload**: 热重载
  - **House**: 房屋系统
  - **MapManager**: 地图管理
  - **PlanOff**: 计划关机
  - **Silent**: 静默模式
  - **duplicateips**: IP 重复检测
  - **koishi**: Koishi 机器人集成
  - **tp-plus**: 传送增强

## 核心功能

### 1. Web 管理面板

- 基于 Vue 3 的现代化管理界面
- 控制台仪表盘：在线玩家、服务器状态实时监控
- 子页面：在线统计、玩家列表、用户详情、服务器设置、文件管理、物品搜索等
- 初始化设置向导（首次启动引导配置）
- 终端控制台（SSE 日志流 + 命令执行）
- 暗色/亮色主题切换

### 2. 反作弊系统

- **物品限制** (`ItemRestrict`): 限制特定物品的使用
- **弹幕限制** (`ProjDetection`): 限制特定弹幕/投射物
- **违禁弹幕管理** (`projlist` 命令): 查看和管理违禁弹幕列表
- **违禁物品管理** (`scanlist` 命令): 查看和管理违禁物品列表
- **自动扫描** (`AutoScan`): 间隔自动扫描玩家背包
- **进度检测** (`AntiCheat`): 基于游戏进度的物品限制

### 3. 玩家管理

- 背包查看/编辑 (`GetPlayerInv`)
- 用户查询/管理 (`QueryUsers`)
- 玩家统计 (`PlayerStats`)
- 密码查询 (`QueryPwd`)
- 角色数据清除 (`ClearCharacter`)
- 封禁/解封操作

### 4. 用户组管理

- 用户组 CRUD (`GroupOP`): 创建、读取、更新、删除用户组
- 权限管理: 添加/移除用户组权限

### 5. 在线统计

- 小时级在线数据统计 (`OnlineData`)
- 在线排行榜
- 玩家在线日历（个人在线记录）
- SSE 日志流（实时日志推送）

### 6. QQ 集成

- QQ 绑定 (`QQBind`): 将游戏账号与 QQ 绑定
- QQ 注册: 通过 QQ 注册游戏账号
- QQ 重置密码: 通过 QQ 重置游戏密码

### 7. 文件管理

- 文件读取/写入 (`FileManager`)
- 目录列表浏览
- 目录树展示
- 文件访问白名单控制

### 8. BOSS 管理

- BOSS 数量限制 (`BossLimit`)
- BOSS 进度查询 (`BossProgress`)
- BOSS 退出控制 (`BossLimitQuit`)
- BOSS 召唤限制 (`BossLimitSummon`)

### 9. 其他功能

- 自动注册 (`AutoRegister`)
- 权限提升配置 (`PromotionManager`)
- 计划关机 (`PlannedOff`)
- BUG 修复 (`BugFixes`)
- 运行时钩子 (`RuntimeHooks`)
- 角色导出 (`ExportPlayer`)
- 密码管理 (`PasswordManager`)
- 未验证玩家管理 (`UnverifiedManager`)
- 权限绕过辅助 (`BypassHelper`)

## 消息格式规范

TSWeb 插件中的控制台消息采用统一格式：

```
[TSWeb] 消息内容
```

示例：
- `[TSWeb] 反作弊配置已重新加载`
- `[TSWeb] 聊天命令已清理`
- `[TSWeb] REST API 路由已清理: X 条`

错误消息格式：
```
[TSWeb] 错误描述: {错误详情}
```
