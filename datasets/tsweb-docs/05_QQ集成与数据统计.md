# QQ 集成与数据统计

## QQ 绑定系统

### 数据库

表名：`qq_bind`

```sql
CREATE TABLE IF NOT EXISTS qq_bind (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    UserId INTEGER NOT NULL UNIQUE,    -- TShock 用户 ID
    QQ TEXT NOT NULL,                   -- QQ 号
    BindTime TEXT NOT NULL DEFAULT (datetime('now', 'localtime')),
    UNIQUE(QQ)
);
```

### REST API

| 端点 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/data/qq/bind` | POST | `data.rest.invsee` | 绑定已有角色到 QQ |
| `/data/qq/register` | POST | `data.rest.invsee` | 创建新角色并绑定 QQ |
| `/data/qq/reset-password` | POST | `data.rest.invsee` | 通过 QQ 号重置密码 |

### 接口详情

#### 1. 绑定 QQ — `BindQQ()`

**入参**：`player`（角色名）, `qq`（QQ 号）

**流程**：
1. 检查角色是否存在 → 不存在返回 404
2. 检查角色是否已绑定 → 已绑定返回 409（显示当前绑定的 QQ）
3. 检查 QQ 是否已被其他角色绑定 → 已绑定返回 409（显示绑定角色名）
4. 插入绑定记录
5. 根据 `PromotionConfig` 配置自动提升权限组（如开启）

**返回**：`{ response: "绑定成功", player, qq }`

#### 2. 注册并绑定 — `RegisterAndBind()`

**入参**：`player`（角色名）, `qq`（QQ 号）

**流程**：
1. 检查 QQ 是否已被绑定 → 已绑定返回 409
2. 检查角色名是否已被注册 → 已注册返回 409（若该角色已绑定 QQ 则显示关联 QQ）
3. 创建新角色（自动生成随机密码）
4. 插入绑定记录
5. 根据配置提升权限组

**返回**：`{ response: "注册并绑定成功", player, qq }`

#### 3. 重置密码 — `ResetPasswordByQQ()`

**入参**：`qq`（QQ 号）, `password`（新密码）

**流程**：
1. 根据 QQ 查找绑定的 UserId → 未绑定返回 404
2. 获取用户账号 → 不存在返回 404
3. 调用 `SetUserAccountPassword()` 修改密码

**返回**：`{ response: "密码修改成功", player, qq }`

### 权限提升

绑定成功后自动触发 `PromotionManager.TryPromote()`，根据 `PromotionConfig.QqBind` 配置的 `TargetGroup` 和 `Mode` 执行权限提升。

## 自动注册系统

### 注册模式

| 模式 | 枚举值 | 说明 |
|------|--------|------|
| Default | `RegisterMode.Default` | 默认行为，允许手动 `/register` |
| Auto | `RegisterMode.Auto` | 自动注册，禁止手动注册命令 |
| Block | `RegisterMode.Block` | 阻止未注册玩家进入 |

### 配置

**文件路径**：`{TShock.SavePath}/TSWeb/config.json`

```json
{
  "SetupCompleted": false,
  "AutoRegisterMode": "default",
  "BossLimitMode": "disabled",
  "BossLimitEnabled": false,
  "BossLimitMinPlayers": 7,
  "QuitLimitEnabled": false,
  "LateCompEnabled": false
}
```

### 自动注册流程 (Auto 模式)

```
玩家连接 → OnGetData(ContinueConnecting2)
├── 已有账号 → 放行（BugFixes 处理 UUID 密码挑战）
└── 无账号
    ├── IP/UUID 被封禁 → 阻止注册
    └── 未封禁
        ├── 创建 UserAccount（随机密码）
        ├── 自动登录（设置组、加载角色数据、恢复 SSC）
        └── 广播欢迎消息
```

### Block 模式流程

```
玩家连接 → OnGetData(ContinueConnecting2)
├── 有账号 → 放行
└── 无账号 → 延迟 200ms 后发送 Disconnect 踢出
```

### 命令

| 命令 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/autoregister` | `/ar` | `tshock.admin` | 查看状态和用法 |
| `/autoregister mode <模式>` | — | `tshock.admin` | 设置注册模式（default/auto/block） |
| `/autoregister reload` | — | `tshock.admin` | 重新加载配置文件 |
| `/autoregister status` | — | `tshock.admin` | 查看当前状态 |

### REST API

| 端点 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/data/config/tsweb` | GET | 公开 | 获取自动注册配置 |
| `/data/config/tsweb/set` | POST | `data.rest.invsee` | 更新自动注册配置 |

**支持设置的参数**：`mode`, `bossLimitMode`, `bossLimitEnabled`, `bossLimitMinPlayers`, `quitLimitEnabled`, `lateCompEnabled`

## 未验证玩家管理

### REST API

| 端点 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/data/users/unverified/list` | GET | `data.rest.invsee` | 获取所有未登录玩家列表 |
| `/data/users/unverified/detail` | GET | `data.rest.invsee` | 获取单个未验证玩家详情 |
| `/data/users/unverified/register` | POST | `data.rest.invsee` | 为未注册玩家创建账号并登录 |
| `/data/users/unverified/force-login` | POST | `data.rest.invsee` | 强制登录已有账号的玩家（更新 UUID） |
| `/data/users/unverified/kick` | POST | `data.rest.invsee` | 踢出未验证玩家 |
| `/data/users/unverified/ban` | POST | `data.rest.invsee` | 封禁未验证玩家（封 IP+UUID） |

### 接口详情

#### 列表 `GET /list`
- 遍历在线玩家，筛选未登录或未注册的玩家
- 返回每个玩家的：nickname, ip, uuid, hasAccount, accountName, isLoggedIn, state, stateText, group

#### 详情 `GET /detail?nickname=`
- 根据昵称查找在线玩家
- 返回与列表相同的完整信息

#### 注册 `POST /register`
- **入参**：nickname, password, group（可选，默认使用 `DefaultRegistrationGroupName`）
- 创建账号 → 强制登录 → 返回成功信息

#### 强制登录 `POST /force-login`
- **入参**：nickname
- 查找玩家账号或数据库账号 → 更新 UUID → 执行强制登录

#### 踢出 `POST /kick`
- **入参**：nickname, reason, character
- 调用 `player.Kick()` 踢出

#### 封禁 `POST /ban`
- **入参**：nickname, reason, character
- 跳过 127.0.0.1 回环 IP
- 封禁 IP + UUID（永久封禁，DateTime.MaxValue）
- 踢出玩家

## 玩家统计（PlayerStats）

### REST API

| 端点 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/data/users/stats` | GET | `data.rest.invsee` | 获取玩家属性数据 |
| `/data/users/stats/set` | POST | `data.rest.invsee` | 修改玩家属性数据 |

### 支持查询/修改的属性

| 字段 | 类型 | 说明 |
|------|------|------|
| health | int | 当前生命值 |
| maxHealth | int | 最大生命值 |
| mana | int | 当前魔力值 |
| maxMana | int | 最大魔力值 |
| questsCompleted | int | 钓鱼任务完成数 |
| extraSlot | bool | 额外配饰栏 |
| unlockedBiomeTorches | bool | 解锁环境火把 |
| ateArtisanBread | bool | 食用工匠面包 |
| usedAegisCrystal | bool | 使用神盾水晶 |
| usedAegisFruit | bool | 使用神盾果实 |
| usedArcaneCrystal | bool | 使用奥术水晶 |
| usedGalaxyPearl | bool | 使用星系珍珠 |
| usedGummyWorm | bool | 使用软糖虫 |
| usedAmbrosia | bool | 使用龙涎香 |
| unlockedSuperCart | bool | 解锁超级矿车 |
| enabledSuperCart | bool | 启用超级矿车 |

### 查询逻辑
1. 优先查询在线玩家内存数据 → `source: "memory"`
2. 离线玩家查询数据库 `tsCharacter` → `source: "database"`
3. 找不到玩家返回 404

### 修改逻辑
1. 先同步在线玩家状态到数据库
2. 逐字段更新（仅修改提交的字段）
3. 在线玩家重新从 DB 加载并同步到客户端

## 在线数据（OnlineData）

### 数据表

```sql
-- 玩家每日在线分钟
player_daily_stat (
    uid TEXT NOT NULL,       -- 玩家名
    date TEXT NOT NULL,      -- 日期 yyyy-MM-dd
    daily_min INTEGER DEFAULT 0,
    PRIMARY KEY (uid, date)
);

-- 每小时在线快照
hourly_online_snapshot (
    hour_ts INTEGER PRIMARY KEY,  -- 时间戳编码 yyyyMMddHH
    online_names TEXT NOT NULL     -- 空格分隔的玩家名列表
);
```

### 数据采集机制

| 采集项 | 触发频率 | 说明 |
|--------|---------|------|
| 每日在线分钟 | 每 60 秒（3600 tick） | 更新 `player_daily_stat`，活跃玩家 +1 分钟 |
| 每小时快照 | 每 60 分钟（216000 tick） | 记录当前所有已登录玩家名到 `hourly_online_snapshot` |
| 每分钟在线更新 | 每 60 秒 | `EnsureHourlyRecord()` 实时更新当前小时记录 |
| 自动权限晋升检查 | 每 60 秒 | 检查玩家游玩时长是否达到晋升阈值（按 `PromotionConfig`） |

### REST API

| 端点 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/data/online/hourly?date=yyyy-MM-dd` | GET | `data.rest.invsee` | 获取指定日期的每小时在线数据 |
| `/data/online/ranking?mode=today/24h/7d/30d/all` | GET | `data.rest.invsee` | 获取在线时长排行榜（Top 100） |
| `/data/online/player?name=&year=` | GET | `data.rest.invsee` | 获取指定玩家指定年份的在线日历 |
| `/data/online/ranking/stats?type=online/deaths/fishing&page=&pageSize=` | GET | `data.rest.invsee` | 通用排行榜（在线/死亡/钓鱼，支持分页） |

### 排行榜类型

| type 参数 | 数据来源 | 说明 |
|-----------|---------|------|
| `online` | `player_daily_stat` | 在线时长排行（分钟） |
| `deaths` | `tsCharacter.deathsPVE` | PVE 死亡次数排行 |
| `fishing` | `tsCharacter.questsCompleted` | 钓鱼任务完成数排行 |

分页参数：`page`（默认1）, `pageSize`（默认10，最大1000）

## SSE 日志系统

### 架构

```
Console 输出 → LogInterceptor (TextWriter)
    ├── 原始输出：写入 Console.Out（保持终端可见）
    └── 结构化解析：按 ConsoleColor 分割为 LogSegment[]
        └── 序列化为 JSON → 添加到环形缓冲区（最大 1000 条）
            └── SSE 客户端推送循环 → HTTP Stream
```

### LogInterceptor

通过 `Console.SetOut()` 劫持控制台输出流，将原始文本按 `Console.ForegroundColor` 分割为带颜色的片段：

```json
[
  {"t": "[14:30:25] ", "c": "Gray"},
  {"t": "[TSWeb] 服务器已启动", "c": "Green"}
]
```

### SSELogger

| 功能 | 说明 |
|------|------|
| 环形缓冲区 | 最多存储 1000 条日志，新日志覆盖旧日志 |
| SSE 流端点 | `GET /data/online/log/stream?token=` |
| 认证 | 通过 query 参数传递 REST Token（EventSource 限制） |
| 推送格式 | `data: [JSON 数组]\n\n` |
| 初始连接 | 立即推送 `data: {"connected":true}\n\n` |
| Socket 剥离 | 通过反射将底层 NetworkStream 与 HttpServer 解耦，防止连接循环误读 SSE 数据 |
| TCP KeepAlive | 开启 KeepAlive（60s/10s）加速断连检测 |

### SSELogCommand

| 端点 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/data/online/log/command` | POST | `data.rest.invsee` | 以 superadmin 身份执行服务器命令 |

**入参**：`cmd`（命令）, `executor`（执行者名，可选）

**行为**：
1. 将命令执行信息写入 Console → LogInterceptor 自动捕获到 SSE
2. 使用 `TSRestPlayer` 以 superadmin 组执行命令
3. 命令输出写入 Console → 自动推送到 SSE 客户端
4. 返回命令输出文本

### 后端路由

| 端点 | 方法 | 认证 | 说明 |
|------|------|------|------|
| `/api/online/log/stream` | GET | query token | SSE 日志流 |
| `/api/online/log/command` | POST | JWT + admin | 执行控制台命令 |
