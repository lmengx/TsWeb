### 入口文件

**`backend/server.js`**（279行）

- 全局错误保护：`uncaughtException` + `unhandledRejection` 防止进程退出
- Express 中间件：`cors()`, `express.json()`
- 请求日志：记录每请求方法、URL、Body
- 静态文件：挂载 `dist/` 目录作为前端静态资源
- 健康检查：`GET /api/health`
- 状态检查：`GET /api/status`（返回 TShock 连接状态）
- IP 地理位置代理：`GET /api/ip-lookup?ip=`（绕过前端 CORS，处理 GBK 编码）
- 前端 SPA 回退：所有未知路由返回 `index.html`

### 启动流程

1. 生成 Setup Token
2. 检查 `config.json` 是否存在
3. **首次启动**（无配置）：端口 3000，自动打开浏览器到 `/setup/intro?token=`
4. **正常启动**：读取配置，测试 TShock 连接，启动 HTTP 服务
5. **端口容错**：端口被占用时自动 +1 重试，最多尝试 10 次
6. 启动控制台：支持 `setup`（打开管理页）、`exit`（退出）命令

### JWT 认证流程

```
客户端登录 → 后端验证 → 签发 JWT Token
后续请求 → Authorization: Bearer <token> 校验
query 参数 token ← SSE 端点专用（EventSource 不支持自定义 Header）
```

### 路由守卫

- **认证检查**：`requiresAuth` 元字段控制登录要求
- **管理员检查**：`requiresAdmin` 元字段控制管理员权限
- **服务器状态检查**：每 10 秒缓存，断连时重定向到 `/error/server`
- **Setup Token 自动登录**：检测 URL 中的 `?token=` 参数，自动登录为 superadmin

## 前后端通信方式

1. **REST API**：标准 JSON 请求/响应，JWT Bearer Token 认证
2. **SSE 流**：`/api/online/log/stream` 用于实时控制台日志推送
3. **后端代理**：前端通过 `/api/tshock/*` 代理访问 TShock REST API

---

## 开发注意事项

- 后端使用 ES Module（`"type": "module"`）
- 监听 `SIGINT` / `SIGTERM` 安全退出
- `uncaughtException` + `unhandledRejection` 全局错误保护防止进程崩溃
- 端口容错机制：端口被占用时自动递增重试（最多10次）
- 前端 Vite 开发模式下 API 自动代理到后端 3000 端口
