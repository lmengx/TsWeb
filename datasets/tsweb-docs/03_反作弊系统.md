# 反作弊系统

## 概述

TSWeb 反作弊系统由**物品检测**、**弹幕检测**和**违规执行器**三大模块组成，支持按**游戏进度**动态匹配违禁规则，提供 REST API 实现配置热重载与远程管理。

---

## 1. AntiCheat.cs — 反作弊核心

### 配置类结构

```csharp
AntiCheatConfig
├── 启用 (bool)                    // 全局开关，默认 true
├── 自动扫描 (bool)               // 定时自动扫描开关，默认 false
├── 自动扫描间隔-秒 (int)         // 默认 600 秒
└── 限制列表 (List<ProgressRestriction>)
    ├── 进度 (string)             // "始终生效" 或 BOSS 名
    └── 限制物品 (List<RestrictedItem>)
        ├── id (int)
        ├── stack (int)           // 限制数量，默认 1
        └── method (string)       // ban / kick / log / 自定义命令
```

### 配置文件路径

| 配置 | 路径 |
|------|------|
| 物品违禁配置 | `{TShock.SavePath}/TSWeb/AntiCheat/物品违禁.json` |
| 弹幕违禁配置 | `{TShock.SavePath}/TSWeb/AntiCheat/弹幕违禁.json` |

### 核心方法

| 方法 | 说明 |
|------|------|
| `Initialize()` | 加载物品配置 + 弹幕配置，输出初始化日志 |
| `LoadConfig()` | 加载物品违禁 JSON，不存在则创建默认配置 |
| `SaveConfig()` | 保存当前物品违禁配置到磁盘 |
| `LoadProjConfig()` | 加载弹幕违禁 JSON，不存在则创建默认配置 |
| `SaveProjConfig()` | 保存弹幕违禁配置到磁盘，保存后自动刷新弹幕缓存 |
| `GetConfig()` | 获取当前物品违禁配置实例 |
| `GetProjConfig()` | 获取当前弹幕违禁配置实例 |
| `GetItemName(int id)` | 根据物品 ID 获取物品名称 |

### 扫描命令

| 命令 | 权限 | 说明 |
|------|------|------|
| `/scan <玩家名>` | `tshock.admin` | 扫描指定玩家（支持在线/离线） |
| `/scan *` | `tshock.admin` | 扫描所有玩家（包括离线玩家数据库） |
| `/scanlist` | `tshock.admin` | 查看当前生效的违禁物品列表 |
| `/projlist` | `tshock.admin` | 查看当前生效的违禁弹幕列表 |

## 2. ViolationExecutor — 违规执行器

违规处理方式支持 4 种模式：

| 模式 | 行为 |
|------|------|
| `ban` | 执行 `/banp` 封禁玩家 + 踢出 |
| `kick` | 执行 `/kick` 踢出玩家 |
| `log` | 仅记录日志，不执行操作 |
| 自定义命令 | 替换占位符 `{playername}`, `{itemid}`, `{itemname}`, `{projid}` 后执行 |

所有违规处理均通过 `Task.Run` 异步执行，防止阻塞主线程。

## 3. ItemDetection.cs — 物品检测（含自动扫描）

### 初始化与生命周期

```csharp
ItemDetection.Initialize()
├── RefreshRestrictedItems()   // 刷新违禁物品缓存
├── StartAutoScan()             // 根据配置启动自动扫描定时器
└── StopAutoScan()              // 停止自动扫描定时器
```

### 检测机制

- **背包扫描**：`ScanOnlinePlayer()` 遍历在线玩家背包，调用 `CheckItem()` 比对违禁列表
- **离线扫描**：`ScanOfflinePlayer()` 读取数据库中的角色数据扫描
- **全服扫描**：`ScanAllPlayers()` 扫描所有在线玩家 + 所有离线玩家

### 检查逻辑流程

```
CheckItem(player, itemId, stack)
├── 全局关闭 → 返回空
├── 玩家有 tshock.admin / tshock.item.usebanned 权限 → 返回空（豁免）
├── 根据 BossProgress 刷新违禁物品缓存
├── 匹配 ID 且 stack ≥ 限制数量 → 加入匹配列表
└── 二次确认（重新刷新缓存后再查一次，防竞态条件）
```

### 自动扫描定时器

- 按配置的 `AutoScanInterval` 秒循环执行
- 每轮扫描所有在线已登录玩家
- 发现违规自动调用 `ViolationExecutor.ExecuteViolation()`

### 丢弃拦截（ItemRestrict）

通过钩子 `GetDataHandlers.ItemDrop` 注册，玩家丢弃违禁物品时：

1. 设置 `e.Handled = true` 阻止掉落
2. 调用 `Main.item[e.ID].TurnToAir()` 清除物品实体
3. 发送 `NetMessage.SendData(21)` 同步客户端
4. 触发违规处理

### 存箱拦截

通过钩子 `GetDataHandlers.ChestItemChange` 注册，玩家向箱子存入违禁物品时：

1. 设置 `e.Handled = true` 阻止存入
2. 刷新箱子槽位恢复客户端显示
3. 发送 `SendData(ChestItem)` 包同步
4. 提示玩家"违禁物品无法存入箱子"
5. 触发违规处理

## 4. ProjDetection.cs — 弹幕检测

### 初始化

```csharp
ProjDetection.Initialize()
└── RefreshRestrictedProjectiles()  // 刷新违禁弹幕缓存
```

### 检测逻辑

```
CheckProjectile(player, type, damage)
├── 全局关闭 → false
├── 玩家有 tshock.admin.projectileban 权限 → false（豁免）
├── 伤害 > 20000 或 < -10 → 异常伤害弹幕，直接 ban
├── 匹配违禁弹幕列表（双重确认刷新）
│   ├── 玩家有 tshock.projectiles.usebanned → 放行
│   └── 违规 → 按配置 method 执行处理
└── 未匹配 → false（放行）
```

### 进度匹配机制

调用 `BossProgress.GetWorldStatus(progress)` 判断当前世界 BOSS 进度是否达到限制条件。只有进度满足时，对应的违禁弹幕/物品规则才生效。

## 5. ItemConfigHandler.cs / ProjConfigHandler.cs — REST 配置 API

### 物品配置 API（ItemConfigHandler）

| 端点 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/data/anticheat/item-config/getitemconfig` | GET | `tshock.admin` | 获取当前物品违禁配置 |
| `/data/anticheat/item-config/saveitemconfig` | POST | `tshock.admin` | 保存物品违禁配置，保存后触发 `ItemDetection.RefreshRestrictedItems()` |
| `/data/anticheat/item-config/scanall` | GET | `tshock.admin` | 全服扫描违禁物品，返回按玩家分组的违规列表 |
| `/data/anticheat/item-config/scan-by-id` | GET | `tshock.admin` | 按物品 ID 扫描所有玩家背包，返回持有者名单 |

### 弹幕配置 API（ProjConfigHandler）

| 端点 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/data/anticheat/proj-config/getprojconfig` | GET | `tshock.admin` | 获取当前弹幕违禁配置 |
| `/data/anticheat/proj-config/saveprojconfig` | POST | `tshock.admin` | 保存弹幕违禁配置，触发 `ProjDetection.RefreshRestrictedProjectiles()` |

### 后端转发 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/anticheat/config` | GET | 获取物品违禁配置（后端代理转发） |
| `/api/anticheat/proj-config` | GET | 获取弹幕违禁配置 |
| `/api/anticheat/proj-config` | POST | 保存弹幕违禁配置 |
| `/api/anticheat/item-config` | GET | 获取物品违禁配置 |
| `/api/anticheat/item-config` | POST | 保存物品违禁配置 |
| `/api/anticheat/check-anomaly` | POST | 检查物品是否异常（id, stack 入参） |

## 6. 重载事件

所有反作弊配置支持通过 TShock 的 `/reload` 命令热重载：

```csharp
GeneralHooks.ReloadEvent += OnReload;

void OnReload() {
    AntiCheat.LoadConfig();                      // 重载物品违禁配置
    AntiCheat.LoadProjConfig();                  // 重载弹幕违禁配置
    ItemConfigHandler.LoadItemConfig();           // 重载物品配置管理器
    ProjDetection.RefreshRestrictedProjectiles(); // 刷新弹幕缓存
    ItemDetection.RefreshRestrictedItems();       // 刷新物品缓存
    ItemDetection.StartAutoScan();               // 重启自动扫描
    PromotionManager.LoadConfig();               // 重载权限提升配置
}
```

## 数据结构定义

### 物品数据类

```csharp
RestrictedItem
├── ID (int)        // 物品 ID
├── Stack (int)     // 限制数量（超过此数量触发处理）
└── Method (string) // 处理方式: ban / kick / log / 自定义

CheatResult
├── PlayerName, PlayerID
├── ItemID, ItemName
├── FoundStack, AllowedStack
├── Progress, Method
└── Slot (int)
```

### 弹幕数据类

```csharp
RestrictedProj
├── ID (int)        // 弹幕 ID
└── Method (string) // 处理方式: ban / kick / log / 自定义

ProjRestriction
├── Progress (string)
└── Projectiles (List<RestrictedProj>)

ProjRestrictionConfig
├── 启用 (bool)
├── 伤害上限 (int)  // 默认 20000
└── 限制列表 (List<ProjRestriction>)
```
