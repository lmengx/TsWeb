# TShock 架构设计文档

## 整体架构

### 三层架构模式

TShock 采用经典的三层架构设计：

```
┌─────────────────────────────────────────┐
│         表示层 (Presentation)            │
│  ┌──────────────────────────────────┐   │
│  │  REST API │ Commands │ CLI       │   │
│  └──────────────────────────────────┘   │
├─────────────────────────────────────────┤
│         业务逻辑层 (Business Logic)      │
│  ┌──────────────────────────────────┐   │
│  │  Modules │ Handlers │ Managers   │   │
│  └──────────────────────────────────┘   │
├─────────────────────────────────────────┤
│         数据访问层 (Data Access)         │
│  ┌──────────────────────────────────┐   │
│  │  DB Managers │ Models            │   │
│  └──────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

## 核心组件架构

### 1. TShock 核心类（TShock.cs）

**职责**：
- 作为 TerrariaPlugin 的主入口
- 管理全局状态和静态资源
- 初始化所有子系统
- 协调各模块工作

**核心属性**：
```csharp
public class TShock : TerrariaPlugin
{
    // 版本信息
    public static readonly Version VersionNum;
    public static readonly string VersionCodename;
    
    // 核心管理器
    public static BanManager Bans;              // 封禁管理
    public static WarpManager Warps;            // 传送点管理
    public static RegionManager Regions;        // 区域管理
    public static BackupManager Backups;        // 备份管理
    public static GroupManager Groups;          // 组管理
    public static UserAccountManager UserAccounts; // 用户管理
    public static ProjectileManagager ProjectileBans; // 投射物禁令
    public static TileManager TileBans;         // 方块禁令
    public static CharacterManager CharacterDB; // 角色管理
    public static SecureRest RestApi;           // REST API
    public static RestManager RestManager;      // REST 管理器
    
    // 数据库连接
    public static IDbConnection DB;
    
    // 配置系统
    public static TShockConfig Config;
    public static ServerSideConfig ServerSideCharacterConfig;
    
    // 玩家数组
    public static TSPlayer[] Players = new TSPlayer[Main.maxPlayers];
}
```

```
1. 客户端连接
   ↓
2. TSPlayer 创建
   ↓
3. 白名单检查
   ↓
4. GeoIP 定位
   ↓
5. 封禁检查
   ↓
6. 权限加载
   ↓
7. SSC 数据加载
   ↓
8. 进入游戏
```

```

   ↓

   ↓
3. 参数解析
   ↓
4. 执行命令
   ↓
5. 数据库更新
   ↓

```

```
1. HTTP 请求
   ↓

   ↓
3. 权限检查
   ↓

   ↓

```
