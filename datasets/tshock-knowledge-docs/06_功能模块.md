# TShock 功能模块文档

## 模块系统概述

### 模块架构
TShock 采用模块化架构，每个功能模块独立封装，便于维护和扩展。

```
┌─────────────────────────────────────┐
│         TShock Core                 │
├─────────────────────────────────────┤
│     Module Manager                  │
├─────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐        │
│  │ Module 1 │  │ Module 2 │  ...   │
│  └──────────┘  └──────────┘        │
└─────────────────────────────────────┘
```

### 模块基类
所有模块继承自 `Module` 基类：
```csharp
public abstract class Module
{
    public virtual void Initialize() { }
    public virtual void Dispose() { }
}
```

## 模块扩展开发

### 创建新模块
```csharp
public class CustomModule : Module
{
    public override void Initialize()
    {
        // 注册命令
        Commands.ChatCommands.Add(new Command("custom", CustomCommand));
        
        // 注册 Hook
        ServerApi.Hooks.GameUpdate.Register(this, OnGameUpdate);
        
        // 初始化数据库表
        SetupDatabase();
    }
    
    public override void Dispose()
    {
        // 清理资源
        Commands.ChatCommands.RemoveAll(c => c.Name == "custom");
        ServerApi.Hooks.GameUpdate.Deregister(this, OnGameUpdate);
    }
    
    private void CustomCommand(CommandArgs args)
    {
        // 自定义命令实现
    }
}
```

### 模块最佳实践
1. **单一职责**: 每个模块只负责一个功能领域
2. **依赖注入**: 使用构造函数注入依赖
3. **事件驱动**: 使用 Hook 系统而非硬编码
4. **资源管理**: 在 Dispose 中清理所有资源
5. **错误处理**: 妥善处理异常，记录日志
6. **配置化**: 使用配置文件而非硬编码

## 模块通信机制

### Hook 系统
```csharp
// 注册 Hook
ServerApi.Hooks.GameUpdate.Register(this, OnGameUpdate);

// Hook 回调
private void OnGameUpdate(EventArgs args)
{
    // 处理游戏更新
}
```

### 事件系统
```csharp
// 定义事件
public class PlayerEventArgs : EventArgs
{
    public TSPlayer Player { get; set; }
}

// 触发事件
public event EventHandler<PlayerEventArgs> PlayerJoined;
PlayerJoined?.Invoke(this, new PlayerEventArgs { Player = player });
```

### 消息总线
```csharp
// 订阅消息
TShock.OnPlayerJoin += OnPlayerJoin;

// 发布消息
TShock.OnPlayerJoin?.Invoke(player);
```
