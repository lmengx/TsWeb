# TShock 配置与部署文档

## 配置文件系统

### 配置文件位置
TShock 的配置文件存储在 `tshock/` 目录下：
```
tshock/
├── config.json              # 主配置文件
├── sscconfig.json           # SSC 配置
├── rest.json                # REST API 配置
├── motd.txt                 # 欢迎消息
├── rules.txt                # 服务器规则
├── whitelist.txt            # 白名单
└── logs/                    # 日志目录
```

### 主配置文件 (config.json)

#### 服务器基本设置

**ServerPort**
- **类型**: Integer
- **默认值**: 7777
- **说明**: Terraria 服务器监听端口
```json
"ServerPort": 7777
```

**MaxSlots**
- **类型**: Integer
- **默认值**: 8
- **说明**: 最大玩家数量
```json
"MaxSlots": 16
```

**ServerName**
- **类型**: String
- **默认值**: "TShock Server"
- **说明**: 服务器名称，显示在服务器列表
```json
"ServerName": "My TShock Server"
```

#### 玩家管理设置

**DefaultGuestGroupName**
- **类型**: String
- **默认值**: "guest"
- **说明**: 未注册玩家的默认组名
```json
"DefaultGuestGroupName": "guest"
```

**DefaultRegistrationGroupName**
- **类型**: String
- **默认值**: "default"
- **说明**: 新注册用户的默认组名
```json
"DefaultRegistrationGroupName": "default"
```

**AllowLoginAnyUsername**
- **类型**: Boolean
- **默认值**: true
- **说明**: 允许用户使用任意用户名登录
```json
"AllowLoginAnyUsername": true
```

**AllowRegisterAnyUsername**
- **类型**: Boolean
- **默认值**: false
- **说明**: 允许注册与角色名不同的用户名
```json
"AllowRegisterAnyUsername": false
```

#### 建筑保护设置

**DisableBuild**
- **类型**: Boolean
- **默认值**: false
- **说明**: 全局禁止建筑
```json
"DisableBuild": false
```

**AllowCutTilesAndBreakables**
- **类型**: Boolean
- **默认值**: false
- **说明**: 允许玩家破坏临时方块（草、罐子等）
```json
"AllowCutTilesAndBreakables": false
```

**AllowIce**
- **类型**: Boolean
- **默认值**: false
- **说明**: 允许在禁止建筑区域放置冰块
```json
"AllowIce": false
```

#### 世界管理设置

**AutoSave**
- **类型**: Boolean
- **默认值**: true
- **说明**: 启用自动保存世界
```json
"AutoSave": true
```

**AnnounceSave**
- **类型**: Boolean
- **默认值**: true
- **说明**: 广播世界保存消息
```json
"AnnounceSave": true
```

**BackupInterval**
- **类型**: Integer
- **默认值**: 10
- **单位**: 分钟
- **说明**: 备份间隔时间
```json
"BackupInterval": 30
```

**BackupKeepFor**
- **类型**: Integer
- **默认值**: 240
- **单位**: 分钟
- **说明**: 备份保留时间（240分钟 = 4小时）
```json
"BackupKeepFor": 2880  // 2天
```


#### 聊天格式设置

**ChatFormat**
- **类型**: String
- **默认值**: "{1}{2}{3}: {4}"
- **占位符**: 
  - {0} = 组名
  - {1} = 组前缀
  - {2} = 玩家名
  - {3} = 组后缀
  - {4} = 聊天消息
```json
"ChatFormat": "[{0}] {1}{2}{3}: {4}"
```

**ChatAboveHeadsFormat**
- **类型**: String
- **默认值**: "{2}"
- **说明**: 头顶聊天显示格式
```json
"ChatAboveHeadsFormat": "{2}"
```

**BroadcastRGB**
- **类型**: Int32[]
- **默认值**: [127, 255, 0]
- **范围**: 0-255
- **说明**: 广播消息颜色（RGB）
```json
"BroadcastRGB": [255, 215, 0]
```

#### 命令设置

**CommandSpecifier**
- **类型**: String
- **默认值**: "/"
- **说明**: 命令前缀符号
```json

```

**CommandSilentSpecifier**
- **类型**: String
- **默认值**: "."
- **说明**: 静默命令前缀符号
```json

```

#### 安全设置

**BCryptWorkFactor**
- **类型**: Integer
- **默认值**: 7
- **范围**: 5-31

```json

```

**BanOnHardcoreDeath**
- **类型**: Boolean
- **默认值**: false
- **说明**: 硬核角色死亡时封禁
```json
"BanOnHardcoreDeath": false
```

**BanOnMediumcoreDeath**
- **类型**: Boolean
- **默认值**: false
- **说明**: 中核角色死亡时封禁
```json
"BanOnMediumcoreDeath": false
```

#### 反作弊设置

**AllowAllowedGroupsToSpawnBannedItems**

**BombExplosionRadius**

#### 日志设置

### SSC 配置文件 (sscconfig.json)

#### 服务器端角色配置

**Enabled**
- **类型**: Boolean
- **默认值**: false
- **说明**: 启用服务器端角色系统
```json
"Enabled": true
```

**ServerSideCharacterSave**
- **类型**: Boolean
- **默认值**: true
- **说明**: 保存服务器端角色数据
```json
"ServerSideCharacterSave": true
```

**StartingInventory**
- **类型**: List<NetItem>
- **默认值**: []
- **说明**: 新玩家初始物品
```json
"StartingInventory": [
  {"NetId": -15, "Prefix": 0, "Stack": 1},  // 铜短剑
  {"NetId": -7, "Prefix": 0, "Stack": 10}   // 火把
]
```

**StartingHealth**
- **类型**: Integer
- **默认值**: 100
- **说明**: 初始生命值
```json
"StartingHealth": 100
```

**StartingMana**
- **类型**: Integer
- **默认值**: 20
- **说明**: 初始法力值
```json
"StartingMana": 20
```

## 部署方式

### 1. Windows 部署

#### 下载与安装
```powershell
# 下载最新版本
# https://github.com/Pryaxis/TShock/releases

# 解压到目标目录
Expand-Archive TShock.zip -DestinationPath C:\TerrariaServer

# 进入目录
cd C:\TerrariaServer
```

#### 启动服务器
```powershell
# 直接运行
.\TShock.Server.exe

# 指定配置文件
.\TShock.Server.exe -config tshock\config.json

# 指定端口
.\TShock.Server.exe -port 7778
```

#### Windows 服务部署
```powershell
# 使用 NSSM 安装为服务
nssm install TShockServer C:\TerrariaServer\TShock.Server.exe
nssm start TShockServer
```

### 2. Linux 部署

#### 系统要求
- .NET 9 Runtime
- Mono 或 .NET Core

#### 安装依赖
```bash
# Ubuntu/Debian
wget https://dot.net/v1/dotnet-install.sh
chmod +x dotnet-install.sh
./dotnet-install.sh --channel 9.0

# CentOS/RHEL
sudo dnf install dotnet-sdk-9.0
```

#### 部署步骤
```bash
# 下载并解压
wget https://github.com/Pryaxis/TShock/releases/download/v5.2/TShock-5.2-linux-x64-Release.zip
unzip TShock-5.2-linux-x64-Release.zip -d /opt/tshock

# 设置权限
chmod +x /opt/tshock/TShock.Server

# 启动服务器
cd /opt/tshock
./TShock.Server
```

#### Systemd 服务配置
```ini
# /etc/systemd/system/tshock.service
[Unit]
Description=TShock Terraria Server
After=network.target

[Service]
Type=simple
User=terraria
WorkingDirectory=/opt/tshock
ExecStart=/opt/tshock/TShock.Server
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable tshock
sudo systemctl start tshock
```

### 3. Docker 部署

#### Dockerfile
```dockerfile
FROM mcr.microsoft.com/dotnet/sdk:9.0 AS build
WORKDIR /app
COPY . .
RUN dotnet publish TShockLauncher -c Release -o out

FROM mcr.microsoft.com/dotnet/runtime:9.0
WORKDIR /app
COPY --from=build /app/out .
COPY --from=build /app/TShockAPI/bin/Release/net9.0 .
EXPOSE 7777
ENTRYPOINT ["dotnet", "TShock.Server.dll"]
```

#### docker-compose.yml
```yaml
version: '3.8'

services:
  tshock:
    build: .
    container_name: tshock-server
    ports:
      - "7777:7777"
      - "7878:7878"  # REST API 端口
    volumes:
      - ./tshock:/app/tshock
      - ./worlds:/app/worlds
    environment:
      - TZ=Asia/Shanghai
    restart: unless-stopped

  mysql:
    image: mysql:8.0
    container_name: tshock-mysql
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: tshock
    volumes:
      - mysql-data:/var/lib/mysql
    restart: unless-stopped

volumes:
  mysql-data:
```

#### 运行 Docker
```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f tshock
```

## 运维管理

### 日志管理

#### 日志文件位置
```
tshock/logs/
├── 2024-01-01_12-00-00.log
├── 2024-01-02_12-00-00.log
└── ...
```

#### 日志级别
- **Info**: 一般信息
- **Warning**: 警告信息
- **Error**: 错误信息
- **Debug**: 调试信息

#### 日志分析
```bash
# 查看最新日志
tail -f tshock/logs/$(ls -t tshock/logs/ | head -1)

# 搜索错误日志
grep -r "ERROR" tshock/logs/

# 统计玩家登录
grep "has joined" tshock/logs/*.log | wc -l
```

### 性能监控

#### 服务器性能指标
- **CPU 使用率**
- **内存使用量**
- **网络带宽**
- **玩家连接数**
- **TPS (Ticks Per Second)**

#### 监控脚本
```bash
#!/bin/bash
# monitor.sh

while true; do
    echo "=== TShock Server Status ==="
    echo "Time: $(date)"
    echo "CPU: $(top -bn1 | grep "Cpu(s)" | sed "s/.*, *\([0-9.]*\)%* id.*/\1/" | awk '{print 100 - $1}')%"
    echo "Memory: $(free -m | awk '/Mem/{printf "%.2f%%", $3/$2*100}')"
    echo "Players: $(netstat -an | grep :7777 | grep ESTABLISHED | wc -l)"
    echo "Uptime: $(uptime -p)"
    echo ""
    sleep 60
done
```

### 备份策略

#### 自动备份脚本
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/backup/tshock"
DATE=$(date +%Y%m%d_%H%M%S)
WORLD_FILE="/opt/tshock/worlds/World.wld"
CONFIG_DIR="/opt/tshock/tshock"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份世界文件
cp $WORLD_FILE "$BACKUP_DIR/world_$DATE.wld"

# 备份配置文件
tar -czf "$BACKUP_DIR/config_$DATE.tar.gz" $CONFIG_DIR

# 备份数据库
cp /opt/tshock/tshock/tshock.sqlite "$BACKUP_DIR/database_$DATE.sqlite"

# 删除7天前的备份
find $BACKUP_DIR -type f -mtime +7 -delete

echo "Backup completed at $DATE"
```

#### 定时任务
```bash
# 每小时备份一次
0 * * * * /opt/tshock/scripts/backup.sh >> /var/log/tshock-backup.log
```

### 故障排查

#### 常见问题

**1. 端口占用**
```bash
# 检查端口占用
netstat -tuln | grep 7777
# 或
ss -tuln | grep 7777

# 结束占用进程
kill -9 $(lsof -t -i:7777)
```

**2. 数据库连接失败**
```bash
# 检查数据库连接
mysql -u user -p -h localhost tshock

# 检查数据库权限
GRANT ALL PRIVILEGES ON tshock.* TO 'tshock'@'localhost';
FLUSH PRIVILEGES;
```

**3. 权限问题**
```bash
# 修复文件权限
chown -R terraria:terraria /opt/tshock
chmod -R 755 /opt/tshock
```

**4. 内存不足**
```bash
# 检查内存使用
free -h

# 增加交换空间
dd if=/dev/zero of=/swapfile bs=1M count=1024
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
```

### 安全加固

#### 1. 防火墙配置
```bash
# UFW 防火墙
sudo ufw allow 7777/tcp  # 游戏端口
sudo ufw allow 7878/tcp  # REST API
sudo ufw enable

# iptables
iptables -A INPUT -p tcp --dport 7777 -j ACCEPT
iptables -A INPUT -p tcp --dport 7878 -j ACCEPT
```

#### 2. 用户隔离
```bash
# 创建专用用户
sudo useradd -r -s /bin/false terraria
sudo chown -R terraria:terraria /opt/tshock
```

#### 3. SSL/TLS 加密
```nginx
# Nginx 反向代理配置
server {
    listen 443 ssl;
    server_name api.yourdomain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location /api/ {
        proxy_pass http://localhost:7878/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 环境变量

```bash

export TSHOCK_DB_TYPE="mysql"
export TSHOCK_DB_HOST="localhost"
export TSHOCK_DB_PORT="3306"
export TSHOCK_DB_NAME="tshock"

# 服务器配置

export TSHOCK_MAX_PLAYERS="16"

# 日志配置
export TSHOCK_LOG_LEVEL="info"
export TSHOCK_LOG_PATH="/var/log/tshock"
```

### 配置优先级
1. 命令行参数（最高优先级）
2. 环境变量
3. 配置文件
4. 默认值（最低优先级）
