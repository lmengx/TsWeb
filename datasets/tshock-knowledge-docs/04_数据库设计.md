# TShock 数据库设计文档

## 数据库概述

### 支持的数据库类型
TShock 支持多种数据库管理系统：
- **SQLite** - 默认选择，轻量级，适合小型服务器
- **MySQL** - 适合中大型服务器，支持分布式部署
- **PostgreSQL** - 企业级选择，支持高级特性

### 数据库架构特点
- ORM 模式：自定义查询构建器
- 连接池：复用数据库连接
- 事务支持：保证数据一致性
- 迁移系统：自动表结构更新

## 核心数据表

### 1. 用户表 (Users)

**表名**: `Users`

**用途**: 存储注册用户信息

**表结构**:
```sql
CREATE TABLE Users (
    ID INTEGER PRIMARY KEY AUTOINCREMENT,
    Username VARCHAR(32) UNIQUE NOT NULL,
    Password VARCHAR(128) NOT NULL,        -- 加密密码
    UUID VARCHAR(64),                      -- 玩家 UUID
    GroupName VARCHAR(32),                 -- 所属组
    LastAccessed DATETIME,                 -- 最后访问时间
    RegisteredAt DATETIME,                 -- 注册时间
    LastIP VARCHAR(64),                    -- 最后登录 IP
    ExtraInfo TEXT                         -- 扩展信息（JSON）
);
```

**字段说明**:
- `ID`: 自增主键
- `Username`: 用户名，唯一索引
- `Password`: 使用 BCrypt 加密的密码
- `UUID`: Terraria 玩家唯一标识符
- `GroupName`: 关联到 GroupList 表
- `LastAccessed`: 用于活跃度统计
- `ExtraInfo`: 存储额外用户数据（如设置、偏好等）

**索引设计**:
```sql
CREATE INDEX idx_users_username ON Users(Username);
CREATE INDEX idx_users_uuid ON Users(UUID);
CREATE INDEX idx_users_group ON Users(GroupName);
```

### 2. 组表 (GroupList)

**表名**: `GroupList`

**用途**: 存储权限组信息

**表结构**:
```sql
CREATE TABLE GroupList (
    GroupName VARCHAR(32) PRIMARY KEY,
    Parent VARCHAR(32),                    -- 父组名称
    Commands TEXT,                         -- 权限列表（逗号分隔）
    ChatColor TEXT,                        -- 聊天颜色（R,G,B）
    Prefix TEXT,                           -- 聊天前缀
    Suffix TEXT                            -- 聊天后缀
);
```

**字段说明**:
- `GroupName`: 组名，主键
- `Parent`: 父组名称，实现权限继承
- `Commands`: 权限节点列表，逗号分隔
- `ChatColor`: RGB 颜色值，如 "255,215,0"
- `Prefix`: 显示在用户名前的前缀
- `Suffix`: 显示在用户名后的后缀

**默认组**:
1. **default** (访客组)
   - 基础权限：登录、注册、聊天
   
2. **registered** (注册用户组)
   - 继承 default 组
   - 额外权限：传送、更改密码等

3. **vip** (VIP 组)
   - 继承 registered 组
   - 额外权限：保留位置、特殊命令等

4. **admin** (管理员组)
   - 管理权限：踢人、封禁、管理等

5. **superadmin** (超级管理员组)
   - 所有权限

### 3. 封禁表 (Bans)

**表名**: `Bans`

**用途**: 存储封禁记录

**表结构**:
```sql
CREATE TABLE Bans (
    TicketNumber INTEGER PRIMARY KEY AUTOINCREMENT,
    Identifier TEXT NOT NULL,              -- 封禁标识（IP或用户名）
    UserID INTEGER,                        -- 关联用户 ID
    IP TEXT,                               -- 封禁 IP
    Reason TEXT,                           -- 封禁原因
    BanningUser TEXT,                      -- 执行封禁的管理员
    DateOfBan DATETIME,                    -- 封禁时间
    Expiration DATETIME,                   -- 过期时间
    BannedOnAccountName TEXT               -- 被封禁的账户名
);
```

**字段说明**:
- `TicketNumber`: 封禁票号，自增主键
- `Identifier`: 封禁标识符（可以是 IP、用户名或 UUID）
- `UserID`: 关联到 Users 表的 ID
- `IP`: 被封禁的 IP 地址
- `Reason`: 封禁原因描述
- `BanningUser`: 执行封禁操作的管理员用户名
- `DateOfBan`: 封禁开始时间
- `Expiration`: 封禁结束时间（NULL 表示永久封禁）
- `BannedOnAccountName`: 被封禁时使用的账户名

**索引设计**:
```sql
CREATE INDEX idx_bans_identifier ON Bans(Identifier);
CREATE INDEX idx_bans_userid ON Bans(UserID);
CREATE INDEX idx_bans_ip ON Bans(IP);
CREATE INDEX idx_bans_date ON Bans(DateOfBan);
```

### 4. 区域表 (Regions)

**表名**: `Regions`

**用途**: 存储保护区域信息

**表结构**:
```sql
CREATE TABLE Regions (
    RegionID INTEGER PRIMARY KEY AUTOINCREMENT,
    RegionName VARCHAR(32) UNIQUE NOT NULL,
    X1 INTEGER NOT NULL,                   -- 区域左上角 X
    Y1 INTEGER NOT NULL,                   -- 区域左上角 Y
    X2 INTEGER NOT NULL,                   -- 区域右下角 X
    Y2 INTEGER NOT NULL,                   -- 区域右下角 Y
    WorldID VARCHAR(64),                   -- 世界标识符
    UserGroups TEXT,                       -- 允许访问的组
    Owner TEXT,                            -- 区域所有者
    IsProtected BOOLEAN DEFAULT 1          -- 是否受保护
);
```

**字段说明**:
- `RegionID`: 区域 ID，自增主键
- `RegionName`: 区域名称，唯一
- `X1, Y1, X2, Y2`: 区域坐标（图块坐标）
- `WorldID`: 关联到世界文件的标识符
- `UserGroups`: 允许在此区域建筑的组列表
- `Owner`: 区域拥有者（用户名）
- `IsProtected`: 是否启用保护

**索引设计**:
```sql
CREATE INDEX idx_regions_name ON Regions(RegionName);
CREATE INDEX idx_regions_world ON Regions(WorldID);
CREATE INDEX idx_regions_owner ON Regions(Owner);
```

CREATE TABLE Warps (

### 6. 角色数据表 (SSC)

**表名**: `SSC`

**用途**: 存储服务器端角色数据

**表结构**:
```sql
CREATE TABLE SSC (
    ID INTEGER PRIMARY KEY AUTOINCREMENT,
    Account INTEGER NOT NULL,              -- 关联用户 ID
    Health INTEGER,                        -- 生命值
    MaxHealth INTEGER,                     -- 最大生命值
    Mana INTEGER,                          -- 法力值
    MaxMana INTEGER,                       -- 最大法力值
    ExtraSlots TEXT,                       -- 额外物品槽
    Inventory TEXT,                        -- 物品栏数据（JSON）
    SpawnX INTEGER,                        -- 出生点 X
    SpawnY INTEGER,                        -- 出生点 Y
    Hair INTEGER,                          -- 发型
    HairColor TEXT,                        -- 发色
    SkinColor TEXT,                        -- 皮肤颜色
    EyeColor TEXT,                         -- 眼睛颜色
    PantsColor TEXT,                       -- 裤子颜色
    ShirtColor TEXT,                       -- 衬衫颜色
    UnderShirtColor TEXT,                  -- 内衣颜色
    ShoeColor TEXT,                        -- 鞋子颜色
    Difficulty INTEGER,                    -- 难度
    PlayTime INTEGER,                      -- 游戏时间（秒）
    FOREIGN KEY (Account) REFERENCES Users(ID)
);
```

**字段说明**:
- 存储完整的玩家角色数据
- 支持所有 Terraria 角色属性
- 使用 JSON 存储复杂数据（物品栏等）
- 外键关联到 Users 表

## 查询构建器架构

### 抽象查询构建器
```csharp
public interface IQueryBuilder
{
    string CreateTable(SqlTable table);
    string Insert(SqlTable table, params SqlColumn[] columns);
    string Update(SqlTable table, params SqlColumn[] columns);
    string Delete(SqlTable table, string where);
    string Select(SqlTable table, string[] columns, string where);
}
```

### 具体实现
- `SqliteQueryBuilder` - SQLite 查询构建器
- `MysqlQueryBuilder` - MySQL 查询构建器
- `PostgresQueryBuilder` - PostgreSQL 查询构建器

### 使用示例
```csharp
// 创建表
var table = new SqlTable("Users",
    new SqlColumn("ID", MySqlDbType.Int32) { Primary = true, AutoIncrement = true },
    new SqlColumn("Username", MySqlDbType.VarChar, 32) { Unique = true }
);

var creator = new SqlTableCreator(db, db.GetSqlQueryBuilder());
creator.EnsureTableStructure(table);

// 插入数据
var query = db.GetSqlQueryBuilder().Insert(table, 
    new SqlColumn("Username", MySqlDbType.VarChar) { Value = "player1" }
);
db.ExecuteNonQuery(query);
```

## 数据备份策略

### SQLite 备份
```bash
# 文件系统备份
cp tshock.sqlite tshock_backup.sqlite

# SQL 备份
sqlite3 tshock.sqlite ".dump" > backup.sql
```

### MySQL 备份
```bash
# mysldump 备份
mysqldump -u user -p tshock > backup.sql

# 定时备份脚本
0 2 * * * mysqldump -u user -p tshock | gzip > /backup/tshock_$(date +\%Y\%m\%d).sql.gz
```

### PostgreSQL 备份
```bash
# pg_dump 备份
pg_dump -U user tshock > backup.sql

# 自定义格式备份
pg_dump -U user -Fc tshock > backup.dump
```

## 安全考虑

### 1. 密码安全
- 使用 BCrypt 加密
- 盐值随机生成
- 不存储明文密码

### 2. SQL 注入防护
- 参数化查询
- 输入验证
- 最小权限原则

### 3. 数据加密
- 敏感数据加密存储
- 传输层加密（SSL/TLS）
- 数据库连接加密

### 4. 访问控制
- 数据库用户权限限制
- 应用层权限验证
- 审计日志记录
