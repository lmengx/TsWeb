# 插件开发指南

## 插件架构概述

TShock 插件基于 **TShock API** 和 **插件库模式** 进行开发。所有插件继承自 `TerrariaPlugin` 基类，通过重写 `Initialize()` 和 `Dispose()` 方法管理插件生命周期。

### 整体架构关系

```
┌─────────────────────────────────────────────────────┐
│                  TShock API                          │
│    ┌─────────────────────────────────────────┐      │
│    │            LazyAPI 基础库                │      │
│    │  (命令系统 / 权限 / Rest API / 配置)      │      │
│    └─────────────────────────────────────────┘      │
│         ↕ 依赖         ↕ 依赖                       │
│    ┌────────────┐  ┌──────────────────┐             │
│    │  linq2db   │  │  MiniGamesAPI    │             │
│    │  (ORM)     │  │  (小游戏框架)     │             │
│    └────────────┘  └──────────────────┘             │
│         ↕                             ↕             │
│    ┌─────────────────────────────────────────┐      │
│    │          各业务插件                        │      │
│    │  AutoReset / Economics / Challenger ...  │      │
│    └─────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────┘
```

### 依赖关系链

- **LazyAPI** 依赖于 **linq2db**（数据库 ORM）
- **Economics.Core** → Economics.RPG / Economics.Shop / Economics.Skill 等子插件
- **MiniGamesAPI** → BuildMaster（建筑大师小游戏）
- **DeltaForce.Core** → DeltaForce.Protocol + LazyAPI + linq2db
- **CaiBotLite** → linq2db + GenerateMap
- **CaiRewardChest** → linq2db + LazyAPI

## LazyAPI 开发框架详解

LazyAPI 是本插件库的核心基础框架，提供了一套完整的开发工具集，包括命令自动注册、权限管理、参数匹配、Rest API 等。

### 1. 自动注册指令

**类声明**

使用 `[Command("指令名")]` 特性标记类，LazyAPI 会在插件实例化阶段自动检测并注册：

```csharp
[Command("test")]
public class TestCommand
{
}
```

**指令方法**

方法必须为 `static` 且 `public`，第一个参数类型必须是 `CommandArgs`：

```csharp
[Command("test")]
public class TestCommand
{
    public static void Run(CommandArgs args)
    {
        args.Player.SendInfoMessage("Hello World!");
    }
}
```

### 2. 子命令系统

方法名默认作为子命令名称。可使用 `[Alias]` 特性添加别名：

```csharp
[Command("test")]
public class TestCommand
{
    // /test run
    public static void Run(CommandArgs args) { }

    // /test go (别名)
    [Alias("go")]
    public static void Execute(CommandArgs args) { }
}
```

使用 `[Main]` 特性标记主命令（无子命令）：

```csharp
[Command("test")]
public class TestCommand
{
    [Main]
    public static void Execute(CommandArgs args) { }
}
```

### 3. 权限管理

使用 `[Permissions]` 特性为指令附加权限，无需手动调用 `HasPermission()`：

```csharp
[Command("test")]
[Permissions("test")]        // 类级别：整个指令集的权限
public class TestCommand
{
    [Permissions("test.use")] // 方法级别：子命令的细粒度权限
    public static void Execute(CommandArgs args) { }
}
```

### 4. 其他内置特性

| 特性 | 说明 |
|------|------|
| `[RealPlayer]` | 阻止非玩家（机器人/控制台）执行指令 |
| `[Flexible]` | 灵活参数匹配，不严格要求参数数量 |
| `[Main]` | 标记为主命令（无子命令） |
| `[Alias("别名")]` | 为子命令添加别名 |

```csharp
[Command("test")]
[Permissions("test")]
public class TestCommand
{
    [Flexible]
    [RealPlayer]
    [Permissions("test.use")]
    public static void Execute(CommandArgs args) { }
}
```

### 5. 参数匹配系统

LazyAPI 支持自动参数匹配和类型转换。方法可声明额外参数，LazyAPI 会自动从 `args.Parameters` 中获取并转换：

```csharp
[Command("test")]
[Permissions("test")]
public class TestCommand
{
    [Permissions("test.use")]
    public static void Execute(CommandArgs args, string name, int age)
    {
        // 使用 /test name 18
        args.Player.SendInfoMessage($"玩家：{name}，年龄：{age}");
    }
}
```

**自动支持的类型：**

| 类型 | 说明 |
|------|------|
| `int` | 整数 |
| `long` | 长整数 |
| `bool` | 布尔值 |
| `string` | 字符串 |
| `TSPlayer` | TShock 玩家对象 |
| `UserAccount` | 用户账户 |
| `DateTime` | 日期时间 |

### 6. Flexible 特性

用于处理页码等参数数量不固定的场景。添加 `[Flexible]` 后，不严格要求参数数量，只要大于方法参数数量即可：

```csharp
[Command("test")]
public class TestCommand
{
    [Flexible]
    [Permissions("test.list")]
    public static void List(CommandArgs args)
    {
        var dataList = GetData();
        args.SendPage(dataList, 1, new PaginationTools.Settings
        {
            MaxLinesPerPage = 30,
            NothingToDisplayString = "当前没有数据",
            HeaderFormat = "数据列表 ({0}/{1})：",
            FooterFormat = "输入 {0}test list {{0}} 查看更多"
        });
    }
}
```

### 7. Rest API

LazyAPI 同样支持 Rest API 的自动注册，与指令系统类似：

```csharp
// 使用相应特性标记 Rest API 处理器
// 参考 ServerTool 中的示例实现
```

| 特性 | 说明 |
|------|------|
| `[RestPath]` | 定义 Rest API 路径 |
| `[Find]` | 查找匹配 |
| `[FindAttribute]` | 查找属性匹配 |

## 插件生命周期

### Initialize 方法

所有插件的入口点，在插件加载时调用。负责：
- 注册命令
- 注册事件钩子
- 加载配置
- 初始化数据库

```csharp
public override void Initialize()
{
    // 注册指令
    Commands.ChatCommands.Add(new Command("permission", CommandMethod, "commandName"));

    // 注册事件
    ServerApi.Hooks.GamePostInitialize.Register(this, OnPostInit);
    ServerApi.Hooks.ServerLeave.Register(this, OnServerLeave);

    // 加载配置
    LoadConfig();
}
```

### Dispose 方法

插件卸载时调用，负责释放资源：
- 注销事件钩子
- 释放数据库连接
- 保存状态

```csharp
protected override void Dispose(bool disposing)
{
    if (disposing)
    {
        // 注销事件
        ServerApi.Hooks.GamePostInitialize.Deregister(this, OnPostInit);
        ServerApi.Hooks.ServerLeave.Deregister(this, OnServerLeave);

        // 保存数据
        SaveData();
    }
    base.Dispose(disposing);
}
```

## 配置系统

### JSON 配置

插件使用 JSON 配置文件，通常位于 `tshock/` 目录下：

```csharp
public class PluginConfig
{
    public bool EnableFeature { get; set; } = true;
    public int Interval { get; set; } = 60;
    public List<string> Messages { get; set; } = new();
}

public class MyPlugin : TerrariaPlugin
{
    private PluginConfig _config;

    private void LoadConfig()
    {
        string path = Path.Combine(TShock.SavePath, "PluginName.json");
        if (File.Exists(path))
        {
            _config = JsonConvert.DeserializeObject<PluginConfig>(File.ReadAllText(path));
        }
        else
        {
            _config = new PluginConfig();
            File.WriteAllText(path, JsonConvert.SerializeObject(_config, Formatting.Indented));
        }
    }
}
```

### 多语言支持

插件支持多语言配置文件，文件名按语言命名：

- `PluginName.json` — 默认配置
- `PluginName.zh-CN.json` — 简体中文
- `PluginName.en-US.json` — 美式英语

LazyAPI 支持 `-culture zh|en` 命令行参数来切换生成配置文件的语言。

**国际化字符串：**

```csharp
using TShockAPI.I18n;

// 在代码中使用 GetString 包装文本
var message = GetString("玩家 {0} 已加入游戏").SFormat(playerName);
```

## 数据库使用（linq2db ORM）

本插件库使用 **linq2db** 作为 ORM（对象关系映射）框架。

### 创建数据库表

```csharp
using LinqToDB;
using LinqToDB.Data;

public class MyPluginDB : DataConnection
{
    public MyPluginDB() : base("connectionString") { }

    public ITable<PlayerData> PlayerData => this.GetTable<PlayerData>();
}

[Table("PlayerData")]
public class PlayerData
{
    [PrimaryKey]
    public int Id { get; set; }
    [Column("PlayerName")]
    public string PlayerName { get; set; }
    [Column("Score")]
    public int Score { get; set; }
}
```

### CRUD 操作示例

```csharp
// 插入
db.PlayerData.Insert(() => new PlayerData
{
    PlayerName = "Player1",
    Score = 100
});

// 查询
var data = db.PlayerData.Where(p => p.PlayerName == "Player1").FirstOrDefault();

// 更新
db.PlayerData.Where(p => p.Id == 1)
    .Set(p => p.Score, 200)
    .Update();

// 删除
db.PlayerData.Where(p => p.Id == 1).Delete();
```

### 数据库表管理

插件在 `Initialize()` 中创建所需的数据库表：

```csharp
public override void Initialize()
{
    // 使用 DataConnection 创建表
    using var db = new MyPluginDB();
    try
    {
        db.CreateTable<PlayerData>();
    }
    catch
    {
        // 表已存在
    }
}
```

## 最佳实践和注意事项

### 1. 代码风格

- 使用 `GetString()` 包装所有用户可见文本以支持多语言
- 方法命名清晰自文档化
- 遵循 C# 命名规范（PascalCase 方法名，camelCase 局部变量）

### 2. 错误处理

- 始终使用 try-catch 包裹可能抛异常的代码
- 避免在事件处理中抛出未捕获的异常
- 使用 TShock.Log 记录错误信息

```csharp
try
{
    // 业务逻辑
}
catch (Exception ex)
{
    TShock.Log.Error($"插件执行错误：{ex.Message}");
}
```

### 3. 性能优化

- 避免在频繁调用的钩子（如 `OnUpdate`）中执行耗时操作
- 使用缓存减少数据库查询
- 及时释放资源（数据库连接、文件流等）

### 4. 配置治理

- 首次加载时生成默认配置文件
- 使用 `JsonIgnore` 特性避免序列化不需要的字段
- 配置变更后提供重载机制

### 5. 插件兼容性

- 检查依赖插件是否已加载
- 使用接口注入而非硬编码引用
- 避免修改 TShock 或其他插件的内部状态

### 6. MiniGamesAPI 开发指南

MiniGamesAPI 提供了小游戏开发的基础框架，包含以下特性：

- **房间管理**：创建、加入、离开、销毁房间
- **玩家管理**：MiniPlayer 类封装，支持无敌模式、创造模式
- **区域管理**：游戏区域定义和检测
- **状态管理**：游戏状态机（等待中、游戏中、评分中）

### 7. 开发者注意事项

- 提交 PR 前请查看仓库 WIKI 中的开发者注意事项
- 插件需要附带 README.md 文档
- 必要时提供 `.pdb` 文件以便调试
- 推荐使用 AutoPluginManager（APM）发布和安装插件

- 使用 `-culture` 命令行参数测试多语言配置

### 9. 重要原则

| 原则 | 说明 |
|------|------|
| **先查源码** | 遇到问题先查阅本地 TShock 和插件源码 |
| **本地优先** | 所有开发资料本地已全，禁止联网搜索 |

| **完整提交** | 每次提交包含完整的实现和 Git 提交信息 |
