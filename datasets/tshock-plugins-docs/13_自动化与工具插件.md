# 自动化与工具插件

## 分类概述

自动化与工具插件为 TShock 服务器提供各类自动化管理和实用工具功能。涵盖世界自动重置、计划任务自动化控制、计时器状态保存、区域指令执行、自动广播、随机广播、进度同步、渔夫任务刷新、旅途物品解锁、Wiki 语言包加载、翻译补丁、Proxy Protocol 支持、ID 输出、地图生成、插件列表导出以及设备判断等。这些插件帮助服主大幅降低手动管理负担，提升服务器运维效率。

## 插件详解

### AutoReset — 完全自动重置

**功能说明**

完全自动重置插件，支持自定义重置内容。可根据击杀次数触发自动重置，重置前后可执行自定义指令和 SQL 命令，支持地图预设和种子设置。

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/reset <种子>` | reset.admin | 重置世界 |
| `/resetdata` | reset.admin | 重置数据（不生成地图） |
| `/rs` | reset.admin | 重置设置 |

**配置要点**

- 配置文件位于 `tshock/AutoReset/AutoReset.zh-CN.json`
- 支持击杀重置（配置生物 ID 和需要击杀次数）
- 重置前后可执行多条指令和 SQL 命令
- 支持替换文件、删除文件
- 支持地图预设（地图名、种子、特殊种子）

**配置示例**

```json5
{
  "替换文件": { "/tshock/原神.json": "原神.json" },
  "击杀重置": {
    "击杀重置开关": false,
    "生物ID": 50,
    "需要击杀次数": 50
  },
  "重置后指令": ["/reload", "/初始化进度补给箱"],
  "重置前指令": ["/结算金币"],
  "重置后SQL命令": ["DELETE FROM tsCharacter"],
  "地图预设": { "地图名": null, "地图种子": "", "特殊种子": 0 }
}
```

### ProgressControls — 计划书

**功能说明**

一个综合性的自动化控制服务器插件。支持自动重置世界、自动重启服务器、自动执行指令、按时间控制 NPC/Boss 进度、删除或复制指定路径文件等。

**指令表**

| 指令 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/pco help` | /pco | 无 | 查看所有帮助指令 |
| `/pco now` | 无 | pco.admin | 同步时间 |
| `/pco delFile` | 无 | pco.admin | 删除指定路径文件或文件夹 |
| `/pco copy` | 无 | pco.admin | 复制文件或文件夹并重命名 |
| `/pco npc help` | 无 | pco.npc | 查看控制NPC进度的指令帮助 |
| `/pco com help` | 无 | pco.com | 查看执行指令的指令帮助 |
| `/pco reload help` | 无 | pco.reload | 查看自动重启的指令帮助 |
| `/pco reset help` | 无 | pco.reset | 查看自动重置的指令帮助 |
| `/pco mess` | /pco view | 任意1个权限 | 查看当前服务器自动化计划 |

**配置要点**

- 配置文件位于 `tshock/ProgressControl.json`
- 支持基于小时数的定时重置、重启和指令执行
- Boss 封禁支持按开服时间逐小时解锁
- 支持 NPC 死亡次数触发指令执行
- 重置前可删除数据库表和文件

### TimerKeeper — 保存计时器状态

**功能说明**

保存电路计时器状态，服务器重启后自动恢复开启计时器。无需任何指令和配置。

**注意事项**

- 如果重置地图，需要删除数据库表 `TimerKeeper` 来重置计时器记录

### SwitchCommands — 区域执行指令

**功能说明**

给"开关"赋予指令执行能力。玩家踩踏或激活开关时自动执行预设指令，支持忽略玩家权限、自定义冷却时间和说明。支持占位符 `$name`（替换为使用开关的玩家名）。

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/switch <添加/列表/删除/冷却/说明/权限忽略/取消/重绑/完成>` | switch.admin | 管理开关指令 |
| `/kg <add/list/del/lq/sm/qxhl/qx/cb/wc>` | switch.admin | 管理开关指令（简写） |
| `/switch ignoreperm <true/false>` | switch.ignoreperm | 设置忽略权限能力 |

**使用流程**

1. 将开关放置在墙壁上
2. 使用 `/kg add <指令>` 绑定指令到开关
3. 使用 `/kg wc` 完成设置
4. 如需无视权限，在第二步前使用 `/kg qxhl`

**配置要点**

- 配置文件位于 `tshock/开关配置表.json`
- 每个开关通过坐标（X, Y）唯一标识
- 可配置指令列表、冷却时间、忽略权限、开关说明

### AutoBroadcast — 自动广播

**功能说明**

定时在服务器中广播消息或执行命令。支持计时广播和触发词广播两种模式。

**配置要点**

- 配置文件位于 `tshock/AutoBroadcast.zh-CN.json`
- 支持计时广播（设置时间间隔）和延迟执行
- 支持触发词语（当玩家发送触发词时触发广播）
- 可设置广播 RGB 颜色
- 可限制广播到指定用户组
- 支持执行服务器指令（消息以 `/` 或 `.` 开头）

**配置示例**

```json5
{
  "广播列表": [
    {
      "广播名称": "示例广播",
      "启用": true,
      "广播消息": ["/say Ciallo～(∠・ω< )⌒★", "自动广播消息"],
      "RGB颜色": [255, 234, 115],
      "时间间隔": 600,
      "延迟执行": 0,
      "广播组": [],
      "触发词语": [],
      "触发整个组": false
    }
  ]
}
```

### RandomBroadcast — 随机广播

**功能说明**

定时从内容表中随机发送一条广播内容（支持执行指令）。支持概率触发和同发数量。

**配置要点**

- 配置文件位于 `tshock/随机广播.json`
- 总概率根据所有内容表的触发概率自动计算
- 消息内容含 `/` 或 `.` 的会当作服务器指令执行
- 支持同发数量（同时发送多组内容）
- 间隔单位精确到 0.1 秒

**配置示例**

```json5
{
  "开启插件": true,
  "同发数量": 1,
  "默认间隔/秒": 1.1,
  "是否开启触发概率": true,
  "内容表": [
    {
      "触发概率": 1,
      "消息内容": [".time 7:30", "我又来啦"],
      "触发颜色": [255.0, 234.0, 115.0]
    }
  ]
}
```

### DataSync — 进度同步

**功能说明**

当两个服务端共用同一个数据库时，实现游戏进度同步。支持所有 Boss、事件和入侵的进度状态同步。

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/重置进度同步` | DataSync | 重置进度 |
| `/进度 [进度名] [true/false]` | DataSync.set | 设置进度 |
| `/进度 local` | DataSync | 查看本地进度 |
| `/进度` | DataSync | 查看进度 |

**REST API**

| 路径 | 权限 | 说明 |
|------|------|------|
| `/DataSync` | DataSync | 查询进度 |

**配置要点**

- 配置文件位于 `tshock/DataSync.json`
- 支持几乎所有 Boss（史莱姆王、克眼、肉山、机械三王、世纪之花、月总等）
- 支持各种事件（血月、日食、南瓜月、霜月、火星暴乱、旧日军团等）

### ReFishTask — 自动刷新渔夫任务

**功能说明**

完成渔夫任务后可无限接取渔夫任务。支持配置是否更换任务鱼。

**配置要点**

- 配置文件位于 `tshock/刷新渔夫任务.json`
- 主要配置项：`是否切换任务鱼`（关闭后可用同一种鱼刷一天）

### WikiLangPackLoader — 加载 Wiki 语言包

**功能说明**

启动服务器后自动设置中文，并加载官方 Wiki 中文语言包以修正翻译（如"克劳得"→"云"）。

**注意事项**

- 使用本插件可能导致一些 RPG 等插件的给物品、BUFF 命令失效
- 语言包来源：Steam 创意工坊（官方 Wiki 中文语言包）
- 由 Wiki 中文编辑团队制作维护

### TransferPatch — 翻译补丁

**功能说明**

通过修改程序集实现翻译配置类，能对大部分插件进行翻译。修改结果永久性，使用前最好备份插件。

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/dump-config <插件名> <类名>` | transferpatch.dump | 导出配置类到文件 |

**配置要点**

- 配置文件位于 `tshock/TransferPatch.json`
- 支持指定目标程序集、目标类和翻译列表
- 支持泛型类和嵌套类的翻译
- 泛型类使用 `\`1`、`2` 表示泛型参数数量

**配置示例**

```json5
{
  "启用": true,
  "执行列表": [
    {
      "目标程序集": "ServerPlugins/TShockAPI.dll",
      "目标类": ["TShockAPI.Configuration.TShockSettings"],
      "翻译列表": {
        "TShockAPI.Configuration.TShockSettings.ServerPassword": "服务器密码",
        "Config/Option": "配置"
      }
    }
  ]
}
```

### ProxyProtocolSocket — 接受 Proxy Protocol 协议

**功能说明**

让 TShock 支持 `proxy protocol` 以获取用户真实 IP。需要搭配开启了 `proxy protocol` 功能的 frp、nginx 等反代软件使用。安装此插件后客户端只能通过反代连接 TShock。

**配置要点**

- 配置文件位于 `tshock/ProxyProtocolSocket.json`
- 可配置日志等级（None/Error/Warning/Info/Debug）
- 可配置超时毫秒数（-1 为不启用超时）

### Platform — 判断玩家设备

**功能说明**

判断玩家使用的游戏设备类型（PE、Stadia、XBOX、PSN、Editor、Switch、PC）。作为前置插件供其他插件调用，可用于聊天前缀、计分板判定等。

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/platform <玩家名>` | platform.use | 查看玩家游玩设备 |

**聊天前缀配置**

可在 `tshock/config.json` 的 `ChatFormat` 中使用占位符 `%platform%` 或 `%device%` 添加设备前缀：

```json5
"ChatFormat": "[%platform%]{1}{2}{3}: {4}"
```
