# 传送与移动插件

## 分类概述

传送与移动插件解决服务器中的玩家移动和快速定位需求：

- **传送请求**：玩家间互相传送，需要对方同意，安全可控
- **死亡回溯**：死亡后快速返回死亡地点，减少跑图痛苦
- **地图传送**：双击大地图直接前往目标位置
- **定位传送**：一键定位并传送到花苞、地牢、神庙、宝藏袋、微光湖等关键位置
- **重生设置**：自定义玩家的出生/重生点位置
- **NPC管理**：快速将城镇 NPC 传送回家
- **建筑传送**：在头顶或指定位置生成已保存的建筑

## TeleportRequest 传送请求

### 概述

经典的 TPA（Teleport Ask）传送请求插件。玩家发起传送请求后，需要目标玩家手动接受或拒绝，支持自动接受/自动拒绝模式。

- **作者**：MarioE，Dr.Toxic，肝帝熙恩
- **依赖**：无（独立 TerrariaPlugin）
- **配置文件**：`tshock/tpconfig.json`

### 指令表

| 语法 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/tpa <玩家>` | 无 | `tprequest.tpat` | 发出传送到指定玩家位置的请求 |
| `/tpahere <玩家>` | 无 | `tprequest.tpat` | 发出把指定玩家传送到你位置的请求 |
| `/接受tp` | `/atp` | `tprequest.gettpr` | 接受传送请求 |
| `/拒绝tp` | `/dtp` | `tprequest.gettpr` | 拒绝传送请求 |
| `/自动接受tp` | `/autoaccept` | `tprequest.tpauto` | 切换自动接受所有人的传送请求 |
| `/自动拒绝tp` | `/autodeny` | `tprequest.tpauto` | 切换自动拒绝所有人的传送请求 |

### 功能特性

- **双向传送**：支持 `tpa`（传送到别人）和 `tpahere`（召唤别人过来）
- **自动模式**：可开启自动接受或自动拒绝，免去每次手动操作
- **请求超时**：发送请求后，定时提醒目标玩家，超时后自动取消
- **TPOverride**：拥有 `tpoverride` 权限可无视对方的传送开关和自动拒绝

### 配置要点

```json
{
  "间隔秒数": 3,       // 每多少秒提示一次目标玩家
  "超时次数": 3         // 提示几次后超时取消（总超时时间 = 间隔秒数 × 超时次数）
}
```

> 总超时时间 = `间隔秒数` × `超时次数`，默认 3×3=9 秒后超时。

### 权限节点

| 权限 | 说明 |
|------|------|
| `tprequest.gettpr` | 接受/拒绝传送请求 |
| `tprequest.tpauto` | 切换自动接受/自动拒绝模式 |
| `tprequest.tpat` | 发起传送请求（tpa/tpahere） |

## Back 返回死亡点

### 概述

记录玩家最近一次死亡的位置，使用指令即可传送回死亡地点，支持冷却时间控制。

- **作者**：Megghy，肝帝熙恩
- **依赖**：LazyAPI
- **配置文件**：`tshock/Back.json`

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/back` | `back` | 返回最后一次死亡的位置 |

### 功能特性

- **自动记录**：玩家每次死亡自动记录死亡坐标
- **冷却控制**：使用 `/back` 后进入冷却，冷却期间无法再次使用
- **退出清空**：玩家退出服务器时自动清空死亡点记录
- **复活检测**：玩家未复活时无法使用 `/back`

### 配置要点

```json5
{
  "Back冷却时间": 20    // 冷却秒数，默认 20 秒
}
```

### 权限节点

| 权限 | 说明 |
|------|------|
| `back` | 使用 /back 返回死亡地点 |

## MapTp 双击大地图传送

### 概述

允许玩家在打开大地图后，双击地图上的任意位置直接传送过去。根据权限不同，可以传送到实心方块位置或仅能传送到非实心方块位置。

- **作者**：Nova4334，肝帝熙恩（汉化适配）
- **依赖**：无（独立 TerrariaPlugin）
- **配置文件**：无

### 功能特性

- **双击传送**：打开大地图（M 键），双击任意位置即可传送
- **智能检测**：检测目标位置是否为实心方块，避免卡入方块
- **双层权限**：普通传送权限仅允许传送到非实心方块位置

### 指令表

| 触发方式 | 权限 | 说明 |
|---------|------|------|
| 大地图双击 | `maptp.noclip` | 传送到目标位置（不包含实心方块） |
| 大地图双击 | `maptp` | 传送到目标位置（包含实心方块） |

### 权限节点

| 权限 | 说明 |
|------|------|
| `maptp.noclip` | 双击传送，禁止传送到实心方块位置 |
| `maptp` | 双击传送，允许传送到实心方块位置（含 noclip 全部功能） |

> **注意**：`maptp` 权限包含 `maptp.noclip` 的全部功能，并额外允许传送到实心方块中。建议普通玩家使用 `maptp.noclip` 权限以避免卡方块。

## DwTP 定位传送

### 概述

使用 `/dw` 或 `/定位` 指令一键传送到微光湖、地牢、神庙、世纪之花苞、宝藏袋等关键位置。通过遍历扫描地图图格实现定位，不依赖配置文件。

- **作者**：羽学
- **依赖**：无（独立 TerrariaPlugin）
- **配置文件**：无

### 指令表

| 语法 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/dw` | `/定位` | `dw.use` | 查看定位传送帮助菜单 |
| `/dw wg` | `/dw 微光` | `dw.use` | 传送到微光湖附近 |
| `/dw sm` | `/dw 神庙` | `dw.use` | 传送到神庙附近 |
| `/dw hb` | `/dw 花苞` | `dw.use` | 传送到世纪之花苞附近 |
| `/dw bag` | `/dw 宝藏袋` | `dw.use` | 传送到最近击败的 Boss 宝藏袋位置 |
| `/dw dl` | `/dw 地牢` | `dw.use` | 传送到地牢附近（骷髅王存活时有效） |

### 功能特性

| 目标 | 实现方式 | 限制 |
|------|---------|------|
| **微光湖** | 扫描全图微光液体图格 | 需要世界中存在微光湖 |
| **神庙** | 扫描丛林神庙砖墙或机关门 | 需要世界中存在神庙 |
| **花苞** | 扫描世纪之花苞图格（ID:238） | 花苞未生长时无法传送 |
| **地牢** | 定位骷髅王（NPC ID:37）或地牢幽魂（NPC ID:438） | 骷髅王已击败后无法定位 |
| **宝藏袋** | 记录击败的最后一个 Boss 死亡位置 | 需要至少击败过一个 Boss |

### 权限节点

| 权限 | 说明 |
|------|------|
| `dw.use` | 使用定位传送指令 |

## BedSet 设置重生点

### 概述

允许玩家通过指令设置自己的出生点，每次死亡或重生时自动传送到设定的位置。数据按玩家名称持久化保存。

- **作者**：pang2333（cmgy）
- **依赖**：LazyAPI
- **配置文件**：`tshock/Bed.json`

### 指令表

| 语法 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/shome` | `/家` | `bed.spawn.set` | 将当前位置设为出生点 |

### 功能特性

- **重生传送**：玩家每次重生后自动传送到设定的出生点（延迟 60 tick ≈ 1 秒）
- **数据持久化**：重生点数据按玩家名称存储，重启服务器不丢失
- **覆盖设置**：多次使用 `/shome` 会覆盖旧的出生点位置

### 配置要点

```json5
{
  "重生点": {
    "玩家名1": { "X": 1234, "Y": 567 },
    "玩家名2": { "X": 2345, "Y": 678 }
  }
}
```

> 配置文件自动管理，玩家通过指令设置后自动写入。

### 权限节点

| 权限 | 说明 |
|------|------|
| `bed.spawn.set` | 设置个人出生点 |

## TownNPCHomes NPC 快速回家

### 概述

一键将所有有家的城镇 NPC 传送回它们的家中。适用于 NPC 被怪物击退、卡在地形中或玩家需要快速访问 NPC 商店时。

- **作者**：棱镜，羽学（优化）
- **依赖**：无（独立 TerrariaPlugin）
- **配置文件**：无

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/npchome` | `tshock.world.movenpc` | 将所有有家的城镇 NPC 传送回家 |

### 功能特性

- **批量传送**：一次性将所有城镇 NPC 传送回家
- **智能过滤**：自动跳过流浪/无家可归的 NPC
- **内存缓存**：使用并发字典缓存 NPC 家的位置，高效处理
- **事件监听**：监听 NPC 搬家事件，自动同步家的位置

### 工作原理

```
1. 玩家输入 /npchome
2. 遍历所有 NPC，筛选 active + townNPC + !homeless 的 NPC
3. 记录每个 NPC 家的坐标（homeTileX, homeTileY）
4. 发送数据包将 NPC 位置更新到家的位置
```

### 权限节点

| 权限 | 说明 |
|------|------|
| `tshock.world.movenpc` | 移动 NPC（TShock 原生权限） |

## CreateSpawn 复制建筑（传送部分）

> 本插件已在 **04_领地与建筑插件.md** 中详细描述，此处仅列出其涉及的传送/位置相关功能。

### 概述

将选定区域的建筑保存为文件，支持在玩家头顶或出生点生成建筑。其传送能力主要体现在 `/cb spawn` 指令的位置计算上。

- **作者**：少司命，羽学，Eustia
- **依赖**：LazyAPI
- **配置文件**：`tshock/CreateSpawn.json`

### 传送相关指令

| 语法 | 权限 | 说明 |
|------|------|------|
| `/cb spawn [名字]` | `create.build.copy` | 在玩家头顶位置生成建筑 |
| `/cb set 1` | `create.build.copy` | 敲击/放置方块设置复制区域左上角 |
| `/cb set 2` | `create.build.copy` | 敲击/放置方块设置复制区域右下角 |

### 传送位置计算

```python
# 玩家使用时：建筑生成在玩家头顶
startX = 玩家.TileX - (建筑宽度 / 2)
startY = 玩家.TileY - 建筑高度

# 控制台使用时：在出生点指定位置生成
startX = 出生点X - 配置.中心X + 配置.微调X
startY = 出生点Y - 配置.计数Y + 配置.微调Y
```

### 配置要点

```json5
{
  "中心X": 0,
  "计数Y": 0,
  "微调X": -21,
  "微调Y": 50
}
```
