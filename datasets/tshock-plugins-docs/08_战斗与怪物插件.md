# 战斗与怪物插件

## 分类概述

战斗与怪物插件主要围绕 Terraria 服务器的战斗系统和怪物管理进行增强。涵盖伤害统计与排名、掉落规则自定义、怪物生成控制、武器修改强化、进度锁管理等方面。这些插件适合**开荒服、RPG 服、竞技服**等需要精细化战斗管理的服务器场景。

## 插件详解

---

### DamageStatistic — 伤害统计

- **作者**: Megghy / Cai
- **依赖**: 无
- **说明**: 在击杀 Boss 后，统计所有玩家对 Boss 造成的伤害并进行播报。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| 无 | 无 | 自动触发，无需指令 |

#### 配置要点

- 无配置文件，即插即用
- 使用 `BossDamageTracker` 重构，自动侦测 Boss 击杀事件
- 支持 i18n 国际化

---

### DamageRuleLoot — 伤害规则掉落

- **作者**: 羽学、西江小子
- **依赖**: 无
- **说明**: 根据玩家对 Boss 造成的输出百分比决定是否掉落宝藏袋，从伤害统计插件基础上进行二创。

#### 指令

| 语法 | 别名 | 权限 | 说明 |
| :-: | :-: | :-: | :-: |
| `/reload` | 无 | `tshock.cfg.reload` | 重载配置文件 |

#### 配置要点

- **低于多少不掉宝藏袋**: 默认 `0.15`（15%），低于此百分比的玩家无法捡取宝藏袋
- **伤害榜播报 / 惩罚榜播报**: 可独立开关
- **自定义转移伤害表**: 支持将攻击特定怪物的伤害转移给关联Boss（如攻击鲨鱼龙给猪鲨造成真伤）
- **参与伤害榜的非BOSS怪ID**: 预设火星飞碟、荷兰飞盗船等特殊处理
- **天顶种子兼容**: 天顶世界新三王统计为美杜莎伤害榜

---

### CriticalHit — 击打提示

- **作者**: White, Stone·Free, 肝帝熙恩
- **依赖**: 无
- **说明**: 攻击NPC产生暴击时，弹出特殊的浮动文字效果，可根据武器类型不同显示不同文字。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| 无 | 无 | 自动触发 |

#### 配置要点

> 配置文件位置：`tshock/CriticalConfig.json`

- **总开关**: 控制插件是否启用
- **仅暴击显示**: `true` 表示仅暴击时显示文字，`false` 则每次攻击都显示
- **消息分类**: 按武器类型（近战、远程、魔法、爆炸）分别配置文字内容和RGB颜色
- 示例：近战可配置"砰!"(255,120,0)、"嘭!"(255,40,50) 等

---

### DeathDrop — 死亡掉落

- **作者**: 大豆子，肝帝熙恩更新优化
- **依赖**: 无
- **说明**: 自定义怪物死亡时的掉落物，支持随机掉落和自定义掉落两种模式，互不影响。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| 无 | 无 | 自动触发 |

#### 配置要点

> 配置文件位置：`tshock/死亡掉落配置表.json`

- **是否开启随机掉落**: 全局随机掉落开关
- **完全随机掉落**: 从 1-5452 号物品中随机选取
- **普通随机掉落物**: 自定义所有怪物共享的随机掉落池
- **随机掉落概率 / 数量**: 可分别配置概率和数量范围
- **自定义掉落设置**: 按怪物ID独立配置掉落规则，支持每个怪物独立设置

---

### DisableMonsLoot — 禁怪物掉落

- **作者**: 羽学
- **依赖**: 无
- **说明**: 通过怪物死亡事件清理当前进度的物品，击败指定怪物后自动关闭清理功能。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| `/kdm` | `killitem.admin` | 查看禁怪掉落指令菜单 |
| `/kdm reset` | `killitem.admin` | 将所有清理开关重置开启 |

#### 配置要点

> 配置文件位置：`tshock/禁怪物掉落.json`

- **插件开关**: 控制插件启用
- **是否全杀**: 需要击败列表中所有怪物才会关闭清理
- **清理范围**: 默认 35（格），掉落物清理检测范围
- **清理列表**: 按怪物ID分组，每组配置怪物ID和对应的物品ID白名单
- 击败组内任意一个怪物ID自动关闭该组的清理功能

---

### BanNpc — 禁止怪物生成

- **作者**: Patrikk / GK
- **依赖**: LazyAPI
- **说明**: 阻止指定怪物在服务器中生成。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| `/bm add [ID]` | `bannpc.use` | 添加禁止生成的NPC ID |
| `/bm del [ID]` | `bannpc.use` | 移除禁止生成的NPC ID |
| `/bm list` | `bannpc.use` | 查看禁止生成表 |

#### 配置要点

> 配置文件位置：`tshock/BanNPC.zh-CN.json`

- **阻止怪物生成表**: JSON数组，存储被禁止的NPC ID列表
- 通过指令动态增删，无需手动编辑配置

---

### MonsterRegen — 怪物进度回血

- **作者**: 途逗、羽学
- **依赖**: 无
- **说明**: 通过计时器给怪物实现每秒回血，击败指定BOSS提升阶级，增加怪物回复量和频率。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| `/reload` | `tshock.cfg.reload` | 重载怪物进度回血配置文件 |

#### 配置要点

> 配置文件位置：`tshock/怪物进度回血.json`

- **开启进度模式**: 击败BOSS提升阶级，提高怪物回复量
- **默认回复血量/百分比**: `0.005`（0.5%），每次回复百分比
- **每次最少/最多回复**: 控制回复量上下限
- **进度BOSS表**: 按怪物ID配置对应阶级数
- **禁止回血表**: 指定不需要回血的怪物ID列表
- *注意: 百分比回血对肉前低血量小怪效果有限*

### ConvertWorld — 击败怪物转换世界

- **作者**: onusai、羽学
- **依赖**: 无
- **说明**: 击败指定怪物后，转换世界所有图格与箱子内物品。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| `/reload` | `tshock.cfg.reload` | 重载配置，自动补充怪物名 |

#### 配置要点

> 配置文件位置：`tshock/击败怪物替换世界物品.json`

- **击败所有**: 需要击败怪物ID内所有怪物才触发转换
- **击杀转换表**: 按怪物ID分组配置转换规则
  - **世界图格替换表**: 将世界中的指定图格ID转换为新图格ID（如肉后将土7→58）
  - **箱子物品替换表**: 将箱子里指定物品ID转换为新物品ID
- 怪物名在使用 `/reload` 时根据怪物ID自动补充

### BossLock — 进度锁

- **作者**: Cai
- **依赖**: 无
- **说明**: 禁止召唤BOSS，按时间控制进度。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| `/bl` | `bosslock` | 显示帮助信息 |
| `/bl list` | `bosslock` | 列出所有定时进度配置 |
| `/bl reset` | `bosslock` | 重置进度配置 |
| `/bl edit` | `bosslock` | 修改进度配置 |
| `/bl unlock` | `bosslock` | 解锁进度 |
| `locklist` | 无 | 查看进度列表 |

#### 配置要点

> 配置文件位置：`tshock/BossLock.json`

```json5
{
  "Locks": [
    {
      "LockSeconds": 43200, // 解锁时间（秒），填-1为不解锁
      "NpcId": 50,          // Boss的NPC ID
      "Name": "史莱姆王"     // 自动生成，无需填写
    }
  ]
}
```

- 每个Boss独立配置解锁倒计时
- 适合作为**PVE开荒服务器**控制游戏进度

### ModifyWeapons — 修改武器

- **作者**: 羽学
- **依赖**: LazyAPI
- **说明**: 修改储存玩家武器数据并自动重读，可使用 `/mw` 指令给予玩家指定属性的物品。

#### 指令

| 语法 | 别名 | 权限 | 说明 |
| :-: | :-: | :-: | :-: |
| `/mw` | 无 | `mw.use` | 指令菜单 |
| `/mw hand` | `/mw h` | `mw.use` | 获取手持物品信息开关 |
| `/mw join` | `/mw j` | `mw.use` | 切换进服重读开关 |
| `/mw list` | `/mw l` | `mw.use` | 列出所有修改物品 |
| `/mw read` | 无 | `mw.use` | 手动重读所有修改物品 |
| `/mw auto` | `/mw at` | `mw.amin` | 自动重读功能开关 |
| `/mw clear` | `/mw cr` | `mw.amin` | 自动清理功能开关 |
| `/mw open 玩家名` | `/mw op` | `mw.admin` | 切换别人进服重读状态 |
| `/mw add 玩家名 次数` | 无 | `mw.admin` | 添加重读次数 |
| `/mw del 玩家名` | 无 | `mw.admin` | 删除指定玩家数据 |
| `/mw set` | `/mw s` | `mw.admin` | 修改自己手持物品属性 |
| `/mw up` | `/mw update` | `mw.admin` | 修改自己手持物品属性（保留原值） |
| `/mw give` | `/mw g` | `mw.admin` | 给指定玩家修改物品并建数据 |
| `/mw all` | 无 | `mw.admin` | 给所有玩家修改物品并建数据 |
| `/mw pw` | `/mw p` | `mw.admin` | 公用武器相关修改 |
| `/mw reads` | `/mw rds` | `mw.admin` | 帮所有人重读或开启进服重读 |
| `/mw reset` | `/mw rs` | `mw.admin` | 重置所有玩家数据 |
| `/reload` | 无 | `tshock.cfg.reload` | 重载配置文件 |
| 无 | 无 | `mw.cd` | 忽略冷却时间与次数的重读武器权限 |

#### 配置要点

> 配置文件位置：`tshock/修改武器.json`

- **初始重读次数**: 默认 2 次
- **自动重读**: 检测到玩家使用经济类指令后自动重读武器
- **公用武器表**: 配置所有玩家可获取的公用武器属性
- **清理修改武器**: 玩家丢出或放入箱子时自动清除修改武器
- **修改属性**: 支持伤害、大小、击退、用速、攻速、弹幕、弹速、弹药、前缀、颜色等

### WeaponPlus — 武器强化（非EC版）

- **作者**: 枳
- **依赖**: 无
- **说明**: 使用游戏内钱币强化武器，支持伤害倍数、击退、大小、攻速、弹速的升级。

#### 指令

| 语法 | 别名 | 权限 | 说明 |
| :-: | :-: | :-: | :-: |
| `/plus help` | 无 | `weaponplus.plus` | 查看武器强化菜单 |
| `/clearallplayersplus` | 无 | `weaponplus.admin` | 清除所有玩家的强化物品 |

#### 配置要点

> 配置文件位置：`tshock/WeaponPlus.json`

- **最多升级次数**: 默认 50 次
- **花费参数 / 升级花费增加**: 控制每次升级所需钱币的递增曲线
- **武器类型区分**: 近战、远程、魔法、召唤、其他分别配置伤害上限和攻速上限
- **各上限倍数**: 伤害上限 1.75~2.5 倍、攻速上限 4~8、击退上限 3.0 倍等
- 数据存储于 `tshock.sqlite` 数据库，不会因丢弃武器或重进服务器丢失
