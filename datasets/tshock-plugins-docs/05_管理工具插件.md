# 管理工具插件

## 分类概述

管理工具插件为服务器管理员提供了一系列运维和管理功能，涵盖日常管理、权限控制、插件维护、命令优化等多个方面：

- **增强管理指令**：在 TShock 原生命令基础上提供更多管理便捷指令
- **权限管理**：批量修改权限、为单个玩家设置独立权限、服主特权配置
- **插件管理**：自动更新/安装/启用/禁用插件、查看已安装插件列表
- **命令优化**：简短指令映射、增强帮助系统
- **运维工具**：控制台执行 SQL、状态文本管理、服务器状态检测

## EssentialsPlus 增强管理指令

### 概述

功能丰富的管理指令扩展插件，在 TShock 原生命令基础上增加了大量实用的管理命令，涵盖查找、冻结、传送、管理等多个维度。

- **作者**：WhiteX / 肝帝熙恩 / Cai
- **依赖**：LazyAPI
- **配置文件**：`tshock/EssentialsPlus.json`

### 指令表

| 语法 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/find <类型> <关键词>` | 无 | essentials.find | 查找物品/命令/图格/墙壁（类型: command/item/tile/wall） |
| `/freezetime` | 无 | essentials.freezetime | 冻结/解冻游戏内时间 |
| `/sethome [名称]` | 无 | essentials.home.delete | 设置家的传送点 |
| `/delhome [名称]` | 无 | essentials.home.delete | 删除已设置的家 |
| `/myhome [名称]` | 无 | essentials.home.tp | 传送到自己的家 |
| `/kickall [原因]` | 无 | essentials.kickall | 踢出所有玩家 |
| `/=` | 无 | essentials.lastcommand | 重复上一次执行的命令 |
| `/more [数量]` | 无 | essentials.more | 将手持物品数量改为指定值 |
| `/moreall` | 无 | essentials.more | 将背包中所有可堆叠物品堆满 |
| `/morefall` | 无 | essentials.more | 将手持物品改为满组（如 999） |
| `/mute add <玩家名>` | 无 | essentials.mute | 将玩家加入禁言名单 |
| `/mute delete <玩家名>` | 无 | essentials.mute | 将玩家移出禁言名单 |
| `/mute help` | 无 | essentials.mute | 查看禁言帮助信息 |
| `/tpallow` | 无 | essentials.tpallow | 允许他人传送到自己身边 |
| `/pvpget` | 无 | essentials.pvp | 强制开启 PvP 模式 |
| `/ruler` | 无 | essentials.ruler | 测量当前位置到敲击点的距离 |
| `/sudo <玩家名> <命令>` | 无 | essentials.sudo | 以目标玩家身份执行命令 |
| `/sudo -force <玩家名> <命令>` | 无 | essentials.sudo.force | 强制以目标玩家身份执行命令 |
| `/timecmd <秒数> <命令>` | 无 | essentials.timecmd | 延迟指定秒数后执行命令 |
| `/timecmd -repeat <秒数> <命令>` | 无 | essentials.timecmd | 按指定秒数间隔重复执行命令 |
| `/back` | 无 | essentials.tp.back | 返回上一个位置（如死亡前位置） |
| `/down` | 无 | essentials.tp.down | 向下传送一段距离 |
| `/left` | 无 | essentials.tp.left | 向左传送一段距离 |
| `/right` | 无 | essentials.tp.right | 向右传送一段距离 |
| `/up` | 无 | essentials.tp.up | 向上传送一段距离 |

### 配置要点

```json
{
  "Pvp禁用命令": [
    "/pvpget"
  ],
  "回退位置历史记录": 5,
  "MySql连接": ""
}
```

| 配置项 | 说明 |
|--------|------|
| `Pvp禁用命令` | 当玩家处于 PvP 状态时禁止使用的命令列表 |
| `回退位置历史记录` | `/back` 命令保存的历史位置数量上限 |
| `MySql连接` | MySQL 数据库连接字符串（留空使用 TShock 默认数据库） |

## AutoPluginManager 自动插件管理器

### 概述

插件自动管理工具，支持从远程源自动安装、升级、启用/禁用插件，支持插件热重载升级。

- **作者**：少司命 / Cai / LaoSparrow
- **依赖**：无
- **配置文件**：`tshock/AutoPluginManager.json`

### 指令表

| 语法 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/apm -l` | `/apm -list` | apm.admin | 列出远程源中所有可用插件 |
| `/apm -i <插件名>` | `/apm -install` | apm.admin | 安装指定插件 |
| `/apm -u <插件名>` | `/apm -upgrade` | apm.admin | 升级指定插件 |
| `/apm -b <插件名>` | `/apm -ban` | apm.admin | 将插件加入排除列表（不更新） |
| `/apm -rb <插件名>` | `/apm -removeban` | apm.admin | 从排除列表中移除插件 |
| `/apm -lb` | `/apm -listban` | apm.admin | 列出排除列表中的插件 |
| `/apm -c` | `/apm -check` | apm.admin | 检查插件更新 |
| `/apm -r` | `/apm -repeat` | apm.admin | 检查插件重复 |
| `/apm -il` | `/apm -locallist` | apm.admin | 列出本地已安装的插件 |
| `/apm -on <插件名>` | 无 | apm.admin | 启用指定插件 |
| `/apm -off <插件名>` | 无 | apm.admin | 禁用指定插件 |

### 配置要点

```json
{
  "允许自动更新": true,
  "插件排除列表": [],
  "热重载升级": true,
  "远程源地址": "http://api.terraria.ink:11434/plugin/get_all_plugins"
}
```

| 配置项 | 说明 |
|--------|------|
| `允许自动更新` | 是否在服务器启动时自动检查并更新插件 |
| `插件排除列表` | 不参与自动更新的插件名称列表 |
| `热重载升级` | 升级后是否尝试热重载（无需重启服务器） |
| `远程源地址` | 插件元数据的远程 API 地址 |

### 权限汇总

| 权限节点 | 说明 |
|----------|------|
| `apm.admin` | 使用 AutoPluginManager 全部功能 |

## Ezperm 批量改权限

### 概述

通过配置文件批量修改组权限的工具，支持为指定组添加或删除权限，适合在大量玩家组中批量部署权限调整。

- **作者**：大豆子 / 肝帝熙恩
- **依赖**：无
- **配置文件**：`tshock/Ezperm.json`

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/inperms` | ezperm.admin | 应用配置文件中定义的组权限修改 |

### 配置要点

```json
{
  "组列表": [
    {
      "组名字": "admin",
      "父组": "",
      "添加的权限": [
        "tshock.admin.*",
        "essentials.*"
      ],
      "删除的权限": []
    },
    {
      "组名字": "vip",
      "父组": "default",
      "添加的权限": [
        "essentials.more",
        "essentials.tpallow"
      ],
      "删除的权限": [
        "tshock.kick"
      ]
    }
  ]
}
```

| 配置项 | 说明 |
|--------|------|
| `组列表` | 需要管理的用户组列表 |
| `组名字` | 目标组的名称 |
| `父组` | 该组继承的父组（留空不继承） |
| `添加的权限` | 需要添加到该组的权限列表 |
| `删除的权限` | 需要从该组移除的权限列表 |

### 使用流程

1. 编辑 `tshock/Ezperm.json` 配置文件
2. 在 `组列表` 中添加需要修改权限的组
3. 填写 `添加的权限`（新增）和 `删除的权限`（移除）
4. 在服务器控制台执行 `/inperms` 应用更改
5. 插件会自动创建不存在的组并应用权限变更

### 权限汇总

| 权限节点 | 说明 |
|----------|------|
| `ezperm.admin` | 使用 `/inperms` 应用配置文件的权限修改 |

## PersonalPermission 玩家单独权限

### 概述

为指定玩家单独添加或删除权限的插件，不依赖用户组体系，通过独立的数据存储为单个玩家设置专属权限。

- **作者**：Megghy / 肝帝熙恩
- **依赖**：无
- **配置文件**：无（数据存储在 `tshock/PersonalPermission.json`）

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/pp add <玩家名> <权限>` | personalpermission.admin | 为指定玩家添加权限 |
| `/pp del <玩家名> <权限>` | personalpermission.admin | 为指定玩家删除权限 |

### 配置要点

```json
{
  "玩家权限列表": {}
}
```

数据以玩家名为键，权限列表为值的 JSON 对象存储。

### 开发者 API

```csharp
// 获取玩家个人权限组
TSPlayer.GetData<Group>("PersonalPermission")
```

该 API 返回一个虚拟的 `Group` 对象，包含该玩家的个人权限列表，开发者可在其他插件中通过此方式读取玩家个人权限。

### 权限汇总

| 权限节点 | 说明 |
|----------|------|
| `personalpermission.admin` | 管理玩家个人权限（添加/删除） |

## ShortCommand 简短指令映射

### 概述

命令别名映射插件，允许管理员将长命令映射为简短易记的别名，支持阻止原始命令和执行冷却功能。

- **作者**：GK / 少司命 / 羽学
- **依赖**：LazyAPI
- **配置文件**：`tshock/ShortCommand.json`

### 功能特性

| 功能 | 说明 |
|------|------|
| **命令映射** | 将原始长命令映射为简短的自定义命令 |
| **阻止原始命令** | 可配置是否禁用被映射的原始命令 |
| **冷却系统** | 为映射命令设置使用冷却时间，防止滥用 |

### 配置要点

```json
{
  "命令表": [
    {
      "原始命令": "/spawn",
      "新命令": "/s",
      "阻止原始": false,
      "冷却": 0
    },
    {
      "原始命令": "/tphere *",
      "新命令": "/tpa",
      "阻止原始": true,
      "冷却": 3
    },
    {
      "原始命令": "/timecmd",
      "新命令": "/tc",
      "阻止原始": false,
      "冷却": 1
    }
  ]
}
```

### 配置字段说明

| 字段 | 说明 |
|------|------|
| `原始命令` | 需要映射的原始命令（可带参数模板，如 `*` 表示任意参数） |
| `新命令` | 映射后的简短命令 |
| `阻止原始` | 是否阻止玩家直接使用原始命令（`true` 为禁止原命令） |
| `冷却` | 命令使用后的冷却时间（秒），`0` 为无冷却 |

## SpclPerm 服主特权

### 概述

服主特权插件（Special Permissions），当指定玩家进入服务器时自动执行预设操作，包括给予无敌、Buff、物品、执行命令、设置 SSC 背包等。

- **作者**：羽学
- **依赖**：无
- **配置文件**：`tshock/SpclPerm.json`

### 功能特性

| 功能 | 说明 |
|------|------|
| **进服给无敌** | 玩家进服时自动开启无敌模式 |
| **进服用命令** | 玩家进服时自动执行一组预设命令 |
| **进服给物品** | 玩家进服时自动给予指定物品 |
| **进服给 Buff** | 玩家进服时自动附加 Buff 效果 |
| **设置 SSC 背包** | 为玩家配置 SSC（服务器端角色）背包内容 |
| **特权名单** | 仅为指定特权玩家执行上述操作 |
| **用户组指定** | 按用户组匹配适用对象 |

### 配置要点

```json
{
  "进服给无敌": false,
  "进服给无敌时间": 0,
  "进服给无敌信息": "你已获得无敌保护！",
  "进服给无敌权限": "spclperm.invincible",
  "命令表": [
    {
      "命令": "/god",
      "执行延迟": 0
    },
    {
      "命令": "/buff 1 3600",
      "执行延迟": 1
    }
  ],
  "物品表": [
    {
      "物品": "月光护身符",
      "数量": 1,
      "前缀": "护佑"
    }
  ],
  "Buff表": [
    {
      "Buff": "铁皮",
      "时间": 3600
    }
  ],
  "特权名单": [
    "玩家名1",
    "玩家名2"
  ],
  "使用用户组": "vip",
  "特权背包表": {
    "物品栏": [],
    "装备栏": [],
    "染料栏": []
  },
  "非特权背包表": {
    "物品栏": [],
    "装备栏": [],
    "染料栏": []
  }
}
```

### 配置字段说明

| 配置项 | 说明 |
|--------|------|
| `进服给无敌` | 是否在玩家进服时给予无敌效果 |
| `进服给无敌时间` | 无敌持续时间（秒），`0` 为永久 |
| `进服给无敌信息` | 给予无敌时的提示消息 |
| `进服给无敌权限` | 触发无敌效果所需的权限 |
| `命令表` | 玩家进服时自动执行的命令列表（支持延迟执行） |
| `物品表` | 玩家进服时自动给予的物品列表 |
| `Buff表` | 玩家进服时自动附加的 Buff 列表 |
| `特权名单` | 指定玩家名，仅这些玩家享受特权（留空则对所有玩家生效） |
| `使用用户组` | 指定用户组，该组所有成员享受特权 |
| `特权背包表` | 特权玩家的 SSC 背包配置 |
| `非特权背包表` | 非特权玩家的 SSC 背包配置 |

### 权限汇总

| 权限节点 | 说明 |
|----------|------|
| `spclperm.invincible` | 进服时自动获得无敌效果 |

## StatusTextManager 模板文本管理器

### 概述

管理 PC 端玩家屏幕右上角状态文本（Status Text）的插件，支持使用占位符插值显示动态信息。

- **作者**：LaoSparrow
- **依赖**：无
- **配置文件**：`tshock/StatusTextManager.json`

### 指令表

| 语法 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/statustext <内容>` | `/st` | statustext.admin | 设置/更新屏幕上的状态显示文本 |

### 动态文本插值表

| 占位符 | 说明 |
|--------|------|
| `{PlayerName}` | 当前玩家名称 |
| `{PlayerGroupName}` | 当前玩家所属用户组名称 |
| `{PlayerLife}` | 当前玩家生命值 |
| `{PlayerMana}` | 当前玩家魔力值 |
| `{OnlinePlayersCount}` | 当前在线玩家数量 |
| `{OnlinePlayersList}` | 当前在线玩家列表 |
| `{WorldName}` | 当前世界名称 |
| `{CurrentTime}` | 游戏内当前时间 |
| `{RealWorldTime}` | 现实世界时间 |
| `{CurrentBiomes}` | 当前所处生物群落 |

### 配置要点

```json
{
  "更新间隔": 1000,
  "默认文本": "欢迎来到服务器！",
  "文本模板": "玩家: {PlayerName}\n在线: {OnlinePlayersCount} 人\n时间: {CurrentTime}\n群系: {CurrentBiomes}",
  "启用自动更新": true
}
```

| 配置项 | 说明 |
|--------|------|
| `更新间隔` | 状态文本更新间隔（毫秒） |
| `默认文本` | 默认显示的状态文本 |
| `文本模板` | 状态文本模板，支持占位符插值 |
| `启用自动更新` | 是否自动刷新状态文本内容 |

### 开发者 API

```csharp
// 监听状态文本更新事件
StatusTextManager.Hooks.OnStatusTextUpdate += (player, text) =>
{
    // 自定义文本处理逻辑
    return text; // 返回修改后的文本
};
```

开发者可以通过 `OnStatusTextUpdate` 钩子自定义状态文本的生成逻辑，在文本显示前对其进行修改。

### 权限汇总

| 权限节点 | 说明 |
|----------|------|
| `statustext.admin` | 使用 `/statustext` 命令管理状态文本 |
