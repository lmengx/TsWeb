# 安全与反作弊插件

## 分类概述

安全与反作弊插件是 Terraria 服务器防御体系的重要组成部分，负责保护服务器免受恶意行为和作弊工具的侵害。涵盖进服认证、代理IP拦截、数据包超时处理、地表弹幕控制、数据包拦截、超进度检测、白名单、区域阻挡、怪物生成控制、客户端修改检测等方面。这些插件适合**公共服务器、竞技服务器、开荒服务器**等需要严格安全管控的场景。

> **关联插件**: 部分功能与 [08_战斗与怪物插件](./08_战斗与怪物插件.md) 中的 `BanNpc` 有交叉；`ServerTools`（位于插件主库）包含客户端修改检测功能。

## 插件详解

---

### Chameleon — 进服前登录

- **作者**: mistzzt、肝帝熙恩修复
- **依赖**: LazyAPI
- **说明**: 首次进服玩家获得提示并要求重新登录，后续进服需输入密码验证，支持 UUID/IP 验证。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| 无 | 无 | 自动触发验证流程 |

#### 配置要点

> 配置文件位置：`tshock/Chameleon.zh-CN.json`

- **验证登录IP是否一致**: 启用后玩家更换 IP 需重新验证
- **等待列表长度**: 同时等待验证的玩家数量
- **强制提示欢迎语**: 自定义欢迎文字
- **验证失败提示语**: 自定义验证失败提示
- **同步注册服务器**: 可配置多个服务器共享账户认证
  - 名称、地址（REST API）、Token
- *注意: 此插件替代了原 TShock 账户系统的交互方式*

---

### Noagent — 禁止代理IP

- **作者**: Jonesn、肝帝熙恩
- **依赖**: 无
- **说明**: 调用第三方 API 检测玩家是否使用代理 IP，拦截代理连接。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| 无 | 无 | 自动检测，无需指令 |

#### 配置要点

- 无配置文件
- 调用 `https://blackbox.ipinfo.app/lookup/` 进行检测
- 无需手动管理，即插即用
- *注意: 依赖外部 API 可用性*

---

### SessionSentinel — 数据包超时处理

- **作者**: 肝帝熙恩
- **依赖**: 无
- **说明**: 踢出长时间不发送数据包的玩家，防止 PE 玩家切后台卡无敌 BUG。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| 无 | 无 | 自动检测，无需指令 |

#### 配置要点

- 无配置文件
- 原理: 检测玩家 20 秒内未发送 `PlayerUpdate` 数据包则踢出
- 主要用于防止 PE 端切后台导致的无敌状态

---

### SurfaceBlock — 禁地表弹幕

- **作者**: 羽学
- **依赖**: LazyAPI
- **说明**: 禁止服务器内玩家在地表使用指定弹幕，防止恶意使用爆炸物破坏地图。

#### 指令

| 语法 | 别名 | 权限 | 说明 |
| :-: | :-: | :-: | :-: |
| `/reload` | 无 | — | 重载配置文件 |
| 无 | 无 | `SurfaceBlock` | 不检测其弹幕是否违规（免检权限） |

#### 配置要点

> 配置文件位置：`tshock/禁地表弹幕.json`

```json5
{
  "启用": true,
  "销毁秒数": 5,
  "物品掉落": false,
  "还原图格": true,
  "是否广播": true,
  "禁用弹幕": [28, 29, 37, 65, 68, ...]
}
```

- **销毁秒数**: 弹幕存在超过此秒数后销毁
- **物品掉落**: 销毁弹幕是否掉落物品
- **还原图格**: 弹幕破坏的图格是否还原
- **禁用弹幕**: 预设了常见破坏性弹幕 ID（爆炸物、挖掘弹幕等）
- 自动适配不同地图大小和种子类型（包括颠倒世界）

### PacketsStop — 数据包拦截

- **作者**: 羽学、少司命
- **依赖**: 无
- **说明**: 拦截指定玩家的数据包，可用于限制游客或违规玩家。

#### 指令

| 语法 | 别名 | 权限 | 说明 |
| :-: | :-: | :-: | :-: |
| `/pksp` | `packetstop` | `packetstop.use` | 指令菜单 |
| `/pksp on` | 无 | `packetstop.use` | 开启数据包拦截 |
| `/pksp off` | 无 | `packetstop.use` | 关闭数据包拦截 |
| `/pksp list` | 无 | `packetstop.use` | 列出拦截名单 |
| `/pksp add 玩家名` | 无 | `packetstop.use` | 添加玩家到拦截名单 |
| `/pksp del 玩家名` | 无 | `packetstop.use` | 从拦截名单移除玩家 |
| `/pksp reset` | 无 | `packetstop.use` | 清空拦截名单 |
| `/reload` | 无 | `tshock.cfg.reload` | 重载配置文件 |

#### 配置要点

> 配置文件位置：`tshock/数据包拦截.json`

```json5
{
  "功能开关": false,
  "拦截玩家名单": ["玩家名"],
  "拦截数据包名": ["ConnectRequest", "PlayerUpdate", ...]
}
```

- 支持拦截多种数据包类型（ConnectRequest、PlayerUpdate、TileSendSquare 等百余种）
- 通过配置文件指定拦截哪些玩家的哪些数据包
- 预设了全部数据包名方便筛选
- 适合**限制违规玩家**或**游客模式**场景

### ProgressRestrict — 超进度检测

- **作者**: 少司命、恋恋魔改三合一
- **依赖**: **DataSync**（前置依赖，需先安装）
- **说明**: 对物品、弹幕、Buff 进行超进度检测，防止玩家跨进度使用高级物品。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| 无 | `progress.item.white` | 超进度物品检查白名单权限 |
| 无 | `progress.projecttile.white` | 超进度弹幕检查白名单权限 |
| 无 | `progress.buff.white` | 超进度 Buff 检查白名单权限 |

#### 配置要点

> 配置文件位置：`tshock/超进度检测.json`

- **惩罚违规**: 对违规玩家施加惩罚
- **惩罚Debuff时长**: 施加负面效果时长
- **公示违规者**: 是否广播违规信息
- **清除违规物品/状态**: 自动清除超进度物品或状态
- **踢出违规玩家**: 可选是否直接踢出
- **限制列表**: 按照游戏进度阶段（史莱姆王→月球领主→各入侵事件）配置限制内容
  - 每个阶段可分别配置限制物品、弹幕、状态的 ID 列表
  - 支持跨服解禁配置
- 进度阶段涵盖：史莱姆王、克苏鲁之眼、邪恶Boss、蜂后、骷髅王、肉山、三王、世纪之花、石巨人、猪鲨、拜月、月总、四柱、事件BOSS等

---

### BetterWhitelist — 白名单

- **作者**: 豆沙、肝帝熙恩修、Cai改
- **依赖**: LazyAPI
- **说明**: 将玩家名字加入白名单，仅白名单内的玩家可进入游戏。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| `/bwl help` | `bwl.use` | 显示帮助信息 |
| `/bwl add {name}` | `bwl.use` | 添加玩家名到白名单 |
| `/bwl del {name}` | `bwl.use` | 将玩家移出白名单 |
| `/bwl list` | `bwl.use` | 显示白名单上的全部玩家 |
| `/bwl true` | `bwl.use` | 启用插件 |
| `/bwl false` | `bwl.use` | 关闭插件 |

#### 配置要点

> 配置文件位置：`tshock/BetterWhitelist.zh-CN.json`

```json5
{
  "白名单玩家": [],
  "插件开关": false,
  "连接时不在白名单提示": "你不在服务器白名单中！"
}
```

- 支持通过指令动态增删白名单玩家
- 支持自定义拒绝提示消息
- 适合**私人服务器**或**限制访问**的场景

### DTEntryBlock — 阻止进入地牢或神庙

- **作者**: 肝帝熙恩
- **依赖**: 无
- **说明**: 阻止玩家在击败骷髅王前进入地牢，阻止在击败世纪之花前进入神庙。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| 无 | `skullking.bypass` | 无视阻止进入地牢 |
| 无 | `Plant.bypass` | 无视阻止进入神庙 |

#### 配置要点

> 配置文件位置：`tshock/阻止进入地牢或神庙.json`

```json5
{
  "阻止玩家进入地牢总开关": true,
  "击杀未击败骷髅王进入地牢的玩家": true,
  "传送未击败骷髅王进入地牢的玩家": true,
  "阻止玩家进入神庙总开关": true,
  "击杀未击败世纪之花进入神庙的玩家": true,
  "传送未击败世纪之花进入神庙的玩家": true
}
```

- 支持两种处理方式: **击杀** 和 **传送**
- 可独立控制地牢和神庙的阻止逻辑
- 支持醉酒世界等特殊种子

---

### BanNpc — 阻止怪物生成

- **作者**: Patrikk / GK
- **依赖**: LazyAPI
- **说明**: 禁止指定怪物在服务器中生成，具有安全防护性质（详见 [08_战斗与怪物插件](./08_战斗与怪物插件.md)）。

#### 指令

| 语法 | 权限 | 说明 |
| :-: | :-: | :-: |
| `/bm add [ID]` | `bannpc.use` | 添加禁止生成的NPC ID |
| `/bm del [ID]` | `bannpc.use` | 移除禁止生成的NPC ID |
| `/bm list` | `bannpc.use` | 查看禁止生成表 |

#### 配置要点

> 配置文件位置：`tshock/BanNPC.zh-CN.json`

- **阻止怪物生成表**: JSON 数组存储被禁止的 NPC ID 列表
- 通过指令动态增删，无需手动编辑配置
- 可用于阻止特定怪物以维护服务器平衡或安全

## 扩展: ServerTools 中的客户端修改检测

`ServerTools` 插件（位于插件主库，依赖 LazyAPI + linq2db）集成了**客户端修改检测**功能，可检测玩家是否使用了修改版客户端。该功能通常包括：

- 检测非法物品修改
- 检测非法属性修改
- 检测客户端版本不一致
- 结合白名单/权限系统灵活控制

> 详细配置请参考 `ServerTools` 插件文档。
