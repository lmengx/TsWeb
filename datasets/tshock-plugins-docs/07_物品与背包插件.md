# 物品与背包插件

## 分类概述

物品与背包类插件围绕玩家物品管理和背包操作展开，覆盖了从基础功能到高级玩法的完整需求链：

- **背包查询**：通过 REST API 查看在线/离线玩家的完整背包数据
- **物品生成**：允许服务器生成正常无法获取的特殊物品
- **离线物品**：玩家离线时可发送物品，上线后领取
- **自动存储**：自动将物品存入存钱罐/保险箱等存储空间
- **物品展示**：手持物品时在聊天和头顶显示物品信息
- **物品不消耗**：指定特定物品使用时不消耗数量
- **背包修改**：管理员直接修改玩家背包内容
- **离线命令**：玩家不在线时预先执行 give 等命令
- **宝箱增强**：无限宝箱、每个玩家独立战利品
- **连锁挖矿**：一次性挖掘相连的同种矿石

## RestInventory REST 查背包接口

### 概述

提供 REST API 接口，可查询在线或离线玩家的完整背包数据，包括生命值、法力值、背包物品、装备、套装、存钱罐、保险箱、虚空袋、护卫熔炉等。

- **作者**：少司命
- **依赖**：无
- **版本**：v1.0.0.7

### REST API

| 路径 | 权限 | 说明 |
|------|------|------|
| `/beanInvsee` | beanInvsee.use | 查询指定玩家背包（支持在线/离线） |

### REST 参数

| 参数 | 说明 |
|------|------|
| `name` | 玩家名称（支持在线和离线账号名） |

### REST 返回数据结构

```json5
{
  "response": "查询成功",
  "data": {
    "OnlineStatu": false,              // 是否在线
    "Username": "玩家名",
    "statLifeMax": 500,                // 最大生命
    "statLife": 500,                   // 当前生命
    "statManaMax": 200,                // 最大法力
    "statMana": 200,                   // 当前法力
    "buffType": [],                    // BUFF ID 数组
    "buffTime": [],                    // BUFF 时间数组
    "inventory": [],                   // 背包 0-58 格
    "miscDye": [],                     // 宠物/坐骑染料
    "miscEquip": [],                   // 宠物/坐骑等
    "Loadout": [],                     // 3 套装备方案
    "trashItem": [],                   // 垃圾桶
    "Piggiy": [],                      // 猪猪存钱罐
    "safe": [],                        // 保险箱
    "Forge": [],                       // 护卫熔炉
    "VoidVault": []                    // 虚空保险箱
  }
}
```

### 配置要点

无配置文件，即装即用。

| 权限 | 说明 |
|------|------|
| beanInvsee.use | 使用 REST 查背包接口 |

## ItemBox 离线物品盒子

### 概述

离线物品盒子系统，管理员可以将物品发送到离线玩家的"邮箱"中，玩家上线时收到提示，使用指令即可领取。支持物品 ID、数量、前缀的完整配置。

- **作者**：Cai & 2409
- **依赖**：无
- **版本**：v2025.05.18.4
- **存储**：SQLite 数据库

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/cbox` 或 `/领盒子` | ItemBox.claim | 领取邮箱中的物品 |
| `/gbox <玩家> <物品ID> <数量> <前缀>` 或 `/给盒子` | ItemBox.give | 给指定玩家发送物品到邮箱 |
| `/rbox` 或 `/重置盒子` | ItemBox.remove | 清空所有玩家的离线背包（管理员用） |

### 功能特性

- **自动提示**：玩家登录时自动检测邮箱是否有物品，并发送提示消息
- **物品图标**：提示和反馈信息中内置物品图标（`[i:itemId]` 格式）
- **自动分堆**：当物品数量超过最大堆叠时，自动拆分到多个格子
- **在线通知**：如果收件人在线，会立即收到邮件通知

### 配置要点

无配置文件，数据通过 SQLite 数据库存储。

### 权限节点

| 权限 | 说明 |
|------|------|
| ItemBox.claim | 领取离线邮箱物品 |
| ItemBox.give | 给玩家发送离线物品 |
| ItemBox.remove | 清空所有离线邮箱数据 |

### 更新日志

| 版本 | 要点 |
|------|------|
| v2025.05.18.4 | 当前版本 |

## AutoStoreItems 自动储存

### 概述

持有指定物品时自动存入存钱罐、保险箱、虚空袋、护卫熔炉等存储空间。支持三种识别模式（自动/手持/装备）和性能模式，还支持自动存钱（硬币堆叠）。

- **作者**：羽学，cmgy雱
- **依赖**：LazyAPI
- **版本**：v1.3.6
- **配置文件**：`tshock/AutoStoreItems.zh-CN.json`

### 指令表

| 语法 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/ast` | `/自存` | AutoStore.use | 指令菜单，查看当前状态 |
| `/ast auto` | `/自存 auto` | AutoStore.use | 开启/关闭自动识别模式 |
| `/ast hand` | `/自存 hand` | AutoStore.use | 开启/关闭手持识别模式 |
| `/ast armor` | `/自存 armor` | AutoStore.use | 开启/关闭装备识别模式 |
| `/ast list` | `/自存 list` | AutoStore.use | 列出自己的自动储存物品表 |
| `/ast clear` | `/自存 clear` | AutoStore.use | 清空自己的自动储存表 |
| `/ast bank` | `/自存 bank` | AutoStore.use | 切换监听存储空间位格开关 |
| `/ast mess` | `/自存 mess` | AutoStore.use | 开启/关闭自动储存消息提示 |
| `/ast add <物品>` | `/自存 add` | AutoStore.use | 添加物品到自动储存表 |
| `/ast del <物品>` | `/自存 del` | AutoStore.use | 从自动储存表移除物品 |
| `/ast pm` | `/自存 pm` | AutoStore.admin | 开启/关闭性能模式 |
| `/astreset` | `/重置自存` | AutoStore.admin | 清空所有玩家自动储存数据 |

### 三种识别模式

| 模式 | 触发条件 | 说明 |
|------|---------|------|
| **自动识别** | 背包或宠物栏中存在"储物类道具" | 玩家移动/攻击时自动触发 |
| **手持识别** | 手持"储物类道具" | 手持特定物品时触发 |
| **装备识别** | 盔甲或饰品栏中存在"装备饰品" | 装备特定饰品时触发 |

> **注意**：三种模式互斥，开启任意一个时其他两个自动关闭。无论哪种模式，都需要玩家**移动或攻击**才会触发储存。

### 配置要点

```json
{
  "插件开关": true,
  "性能模式": true,
  "存钱罐": true,
  "保险箱": true,
  "虚空袋": true,
  "护卫熔炉": true,
  "触发存储的物品ID": [
    87, 346, 3213, 3813, 4076, 4131, 5098, 5325
  ],
  "装备饰品的物品ID": [
    88, 410, 411, 489, 490, 491, 855, 935, 1301, 2220,
    2998, 3034, 3035, 3061, 3068, 4008, 4056, 4989, 5098, 5107, 5126
  ]
}
```

> **存储空间说明**：`存钱罐(87)`、`保险箱(346)`、`虚空袋(3213/3813)`、`护卫熔炉(4076/4131/5098/5325)`

### 性能模式说明

- **开启时**：堆叠达到单格上限（如 9999）的物品不再进行分堆累积
- **关闭时**：会为超堆叠的物品在存储空间中查找空槽继续累积
- 建议：服务器人少时可关闭以获得更好的自存体验

### 权限节点

| 权限 | 说明 |
|------|------|
| AutoStore.use | 使用自动储存功能 |
| AutoStore.admin | 管理自动储存（重置/性能模式） |

## ItemPreserver 物品不消耗

### 概述

指定特定物品在使用时不会消耗数量。可配置需要物品数量达到什么条件才触发不消耗，适合用于稀有/特殊物品的保护。

- **作者**：肝帝熙恩，少司命
- **依赖**：无
- **版本**：v1.0.11
- **配置文件**：`tshock/ItemPreserverConfig.json`

### 功能特性

- 按物品 ID 配置不消耗规则
- 支持数量条件（达到指定数量后才开始不消耗）
- `-1` 表示无条件不消耗
- 通过快捷键使用的物品可能仍会消耗（如快捷喝药）

### 配置要点

```json5
{
  "不消耗物品": {
    "25": -1,     // 物品ID 25（如琥珀）: -1 代表不需要条件，直接不消耗
    "1145": 30    // 物品ID 1145: 需要至少 30 个时才不消耗
  }
}
```

> **格式说明**：`"物品ID": 所需数量`，`-1` 表示不设条件，有该物品即可无限使用。

### 权限节点

无（自动生效）

## RolesModifying 修改玩家背包

### 概述

管理员可以查看和修改在线或离线玩家的完整背包数据。支持复制玩家的背包、装备、饰品、存钱罐、保险箱等所有数据到管理员身上，修改完毕后保存生效。

- **作者**：少司命
- **依赖**：无
- **版本**：v1.0.7

### 指令表

| 语法 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/rm <玩家名>` | `/修改`、`/查背包` | tshock.user.rolesmodifying | 查看并修改指定玩家的背包 |
| `/rm save` | - | tshock.user.rolesmodifying | 保存对玩家背包的修改 |

### 使用流程

```bash
# 1. 查看目标玩家背包（复制其数据到管理员身上）
/rm 目标玩家名

# 2. 此时管理员的背包变成了目标玩家的背包，可以自由修改
# （如丢弃物品、放入新物品等）

# 3. 修改完成后保存
/rm save
```

### 功能特性

- **支持在线玩家**：直接复制在线玩家的完整角色数据
- **支持离线玩家**：从数据库读取离线玩家的存档数据
- **完整数据复制**：背包、盔甲、染料、饰品、宠物、存钱罐、保险箱、虚空袋、护卫熔炉、3 套装备方案、垃圾桶等
- **自动恢复**：管理员退出时自动恢复自身数据
- **数据持久化**：保存修改到数据库

### 配置要点

无配置文件。

### 权限节点

| 权限 | 说明 |
|------|------|
| tshock.user.rolesmodifying | 使用背包修改功能 |

## CGive 离线命令系统

### 概述

当目标玩家不在线时，预先添加 give 等命令，待玩家上线时自动执行。支持针对单个玩家（personal 模式）或所有玩家（global 模式，`-1`），还支持 `{name}` 占位符自动替换为玩家名。

- **作者**：Leader，肝帝熙恩
- **依赖**：无
- **版本**：v1.0.1.1
- **存储**：SQLite 数据库

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/cgive add <执行者> <被执行者> <命令>` | cgive.admin | 添加离线命令 |
| `/cgive list` | cgive.admin | 列出所有离线命令 |
| `/cgive del <id>` | cgive.admin | 删除指定 ID 的离线命令 |
| `/cgive reset` | cgive.admin | 重置所有离线命令数据 |

### REST API

| 路径 | 说明 |
|------|------|
| `/getWarehouse` | 查询玩家的仓库物品（通过分析 give 命令提取物品信息） |

#### REST 参数

| 参数 | 说明 |
|------|------|
| `name` | 玩家注册名 |

#### REST 返回示例

```json5
{
  "response": "获取成功",
  "data": [
    { "stack": 50, "type": 666, "prefix": 0 }
  ]
}
```

### 命令格式详解

```bash
# 语法
/cgive add <执行者> <被执行者> <命令>

# 参数说明：
#   执行者   : Server 或 玩家名（谁的名义执行命令）
#   被执行者  : 玩家名（精确匹配）或 -1（所有玩家）
#   命令     : 要执行的指令，{name} 会被替换为玩家名

# 示例 1：给指定离线玩家发物品
/cgive add Server Steve /give 666 50

# 示例 2：所有玩家上线时执行
/cgive add Server -1 /give 666 1

# 示例 3：使用 {name} 占位符
/cgive add Server PlayerA /say 欢迎 {name} 回来！
```

### 功能特性

- **Personal 模式**：指定玩家名，仅该玩家上线时执行（精确匹配）
- **Global 模式**：`who=-1`，所有玩家上线时都执行，且每个玩家只执行一次
- **{name} 占位符**：命令中的 `{name}` 自动替换为当前上线的玩家名
- **REST 仓库查询**：通过 `/getWarehouse` 接口查询玩家的仓库物品列表

### 配置要点

无配置文件，数据通过 SQLite 数据库存储。

## ChestRestore 无限宝箱

### 概述

实现资源服最常见的"无限宝箱"功能：全局宝箱中的物品无限拿取且不可更改内容。同时支持禁止箱子改名，管理员可通过指令切换修改模式来编辑箱子。

- **作者**：Cjx 重构，肝帝熙恩简单修改
- **依赖**：无
- **版本**：v1.2.0

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/chestedit` 或 `/ce` 或 `/修改箱子` | chest.edit | 切换个人修改箱子名字和内容的模式 |
| `/mc` | chest.edit | 将所有箱子中的物品填充为最大堆叠数量 |
| 无（自动） | chest.name | 拥有此权限的玩家可直接修改箱子名字 |

### 功能特性

- **无限物品**：箱子中的物品始终可拿取，不会被真正消耗
- **禁止改名**：默认禁止所有玩家修改箱子名称
- **编辑模式**：使用 `/ce` 进入修改模式后，方可编辑箱子内容和名称
- **批量填充**：使用 `/mc` 一键将世界所有箱子的物品堆叠填满至最大值
- **自动恢复**：玩家离开时自动退出编辑模式

### 配置要点

无配置文件，即装即用。

### 权限节点

| 权限 | 说明 |
|------|------|
| chest.name | 允许修改箱子名称 |
| chest.edit | 进入箱子编辑模式，使用 `/mc` 填充箱子 |

## PerPlayerLoot 玩家独立战利品

### 概述

为每个战利品箱创建独立于其他玩家的库存，确保每位玩家打开箱子时都能获得专属的战利品，即使其他人已经开过同一箱子。适合解决地图资源不足问题。

- **作者**：Codian，肝帝熙恩汉化
- **出处**：[GitHub - xxcodianxx/PerPlayerLoot](https://github.com/xxcodianxx/PerPlayerLoot)
- **依赖**：无
- **版本**：v2.0.6
- **数据库**：`tshock/perplayerloot.sqlite`

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/ppltoggle` | perplayerloot.toggle | 全局切换是否启用独立战利品箱 |

> **警告**：`/ppltoggle` 切换可能导致 `Main.chest` 数组与插件内部状态不同步，不推荐在正常游戏环境中使用。仅限调试目的。

### 功能特性

- **每个玩家独立箱子数据**：不同玩家打开同一个世界宝箱，看到的内容各自独立
- **自动创建**：玩家首次打开宝箱时自动创建其专属副本
- **玩家放置的箱子豁免**：玩家自行放置的箱子不受影响，显示真实库存
- **数据持久化**：世界保存时自动保存假宝箱数据

### 使用说明

- **最佳安装时机**：服务器开启之初即安装本插件。若在游戏过程中安装，所有现有箱子将被视为战利品箱，其库存将为每位玩家复制
- **玩家放置的箱子**：在安装了本插件后放置的箱子会被自动识别为"玩家放置"，不受独立战利品影响

### 配置要点

无配置文件，数据通过 SQLite 数据库存储在 `tshock/perplayerloot.sqlite`。

### 权限节点

| 权限 | 说明 |
|------|------|
| perplayerloot.toggle | 切换独立战利品箱功能 |

## VeinMiner 连锁挖矿

### 概述

一次性挖掘所有相连的同种矿石，支持配置可连锁的矿石类型、忽略表面方块和奖励规则。挖掘的矿石自动放入背包或在达到条件时替换为指定奖励物品。

- **作者**：Megghy, YSpoof, Maxthegreat99, 肝帝熙恩, Cai
- **出处**：[GitHub - Maxthegreat99/TSHockVeinMiner](https://github.com/Maxthegreat99/TSHockVeinMiner)
- **依赖**：无
- **版本**：v1.6.1.1
- **配置文件**：`tshock/VeinMiner.json`

### 指令表

| 语法 | 权限 | 说明 |
|------|------|------|
| `/vm` | veinminer | 开关连锁挖矿功能 |
| `/vm <任意参数>` | veinminer | 开关连锁挖矿的消息提示（不影响挖矿功能） |

### 配置要点

```json5
{
  "启用": true,
  "放入背包": true,
  "矿石物块ID": [
    6,    // 铜矿
    7,    // 铁矿
    8,    // 银矿
    9,    // 金矿
    10,   // 锡矿
    11,   // 铅矿
    12,   // 钨矿
    13,   // 铂金矿
    37,   // 陨铁
    56,   // 魔矿
    57,   // 猩红矿
    75,   // 黑曜石
    58,   // 狱石
    107,  // 钴矿
    108,  // 钯金矿
    111,  // 秘银矿
    166,  // 精金矿
    221,  // 钛金矿
    1106, // 夜明矿
    168,  // 沙漠化石
    180,  // 裸露宝石
    75,   // 黑曜石
    180,  // 蓝宝石
    181,  // 红宝石
    182,  // 祖母绿
    183,  // 黄玉
    184,  // 紫水晶
    185   // 钻石
  ],
  "忽略挖掘表面方块ID": [],
  "奖励规则": []
}
```

### 奖励规则配置

```json5
{
  "奖励规则": [
    {
      "仅给予物品": true,           // true=不挖掘矿石，仅给予奖励
      "最小尺寸": 10,               // 连锁矿石达到此数量才触发
      "矿石物块ID": 168,            // 目标矿石类型（沙漠化石）
      "奖励物品": {
        "666": 1,                   // 奖励 1 个物品 666
        "669": 1                    // 奖励 1 个物品 669
      }
    }
  ]
}
```

### 功能特性

- **连锁挖掘**：八方向递归搜索相连的同种矿石，最大连锁块数 5000
- **奖励规则**：当连矿石达到指定大小时，可替换为自定义奖励物品
- **表面方块忽略**：配置表面方块 ID，当矿石上方有指定方块时不会连锁（防刷矿）
- **放入背包**：挖掘的矿石自动进入玩家背包
- **背包保护**：背包满时提示玩家，防止刷物品
- **消息开关**：可单独开关连锁挖矿的提示消息
