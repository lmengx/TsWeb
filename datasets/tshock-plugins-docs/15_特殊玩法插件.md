# 特殊玩法插件

## 分类概述

特殊玩法插件为 TShock 服务器带来全新的游戏模式和玩法体验。包括"三角洲行动"系列玩法模组（在 Terraria 中实现类似三角洲部队的战术射击体验）、挑战者模式（增强游戏难度的全套装备和Boss魔改）、以及建筑大师小游戏（速建大比拼）。这些插件彻底改变了原版的游戏方式，为玩家提供全新的游戏目标。

## 插件详解

### DeltaForce 系列 — 三角洲行动玩法模组

**系列概述**

三角洲行动是一个完整的玩法模组，在 Terraria 中实现类似"三角洲部队"或"逃离塔科夫"风格的战术射击游戏体验。由三个子插件组成：

- **DeltaForce.Protocol**：通信协议库，定义了服务器之间的数据包格式
- **DeltaForce.Core**：特勤处（匹配大厅），负责玩家匹配、装备管理、交易行和哈夫币系统
- **DeltaForce.Game**：游戏服务器，负责实际游戏逻辑（出生点、死亡掉落、撤离点、箱子分配）

> ⚠️ 必须配合 [RealmNexus（维度）](https://github.com/UnrealMultiple/RealmNexus) 使用，否则无法传送玩家

---

### DeltaForce.Protocol — 通信协议库

**功能说明**

定义特勤处服务器和游戏服务器之间的所有通信数据包格式和处理逻辑。采用二进制序列化，基于 TCP 协议通信。

**数据包类型**

```csharp
public enum PacketType : byte
{
    Request, Response,
    Connect, Disconnect,
    GameState, GameStateResponse,
    Heartbeat, HeartbeatResponse,
    PlayerInventoryRequest, PlayerInventoryResponse,
    SaveInventoryRequest, SaveInventoryResponse,
    SquadDataRequest, SquadDataResponse,
    ItemListRequest, ItemListResponse,
    ClientIdentity, ClientIdentityResponse,
    PlayerPositionRequest, PlayerPositionResponse,
}
```

**主要数据包**

| 数据包 | 方向 | 说明 |
|--------|------|------|
| GameStatePacket | Core→Game | 通知游戏开始/结束 |
| ItemListPacket | Game↔Core | 请求/返回物品配置列表 |
| SquadDataPacket | Game↔Core | 请求/返回小队数据 |
| PlayerInventoryPacket | Game↔Core | 请求/返回玩家背包数据 |
| SaveInventoryPacket | Game→Core | 保存玩家背包数据 |
| HeartbeatPacket | 双向 | 维持连接和检测断开 |
| ClientIdentityPacket | Game→Core | 客户端身份验证 |

**使用示例**

```csharp
// 发送请求
var request = new ItemListRequestPacket();
var response = await client.RequestAsync<ItemListRequestPacket, ItemListResponsePacket>(request);

// 处理请求
public class ItemListHandler : RequestHandlerBase<ItemListRequestPacket, ItemListResponsePacket>
{
    public override ItemListResponsePacket Handle(ItemListRequestPacket request)
    {
        var items = GetItemsFromConfig();
        var response = CreateSuccessResponse(request, "Success");
        response.Items = items;
        return response;
    }
}
```

**序列化特性**

```csharp
[PacketType(PacketType.GameState)]
public class GameStatePacket : IRequestPacket
{
    public Guid RequestId { get; set; }
    public bool IsGameActive { get; set; }
    public int GameMode { get; set; }
    public int GameTime { get; set; }
}
```

**依赖**

- .NET 9.0
- 无外部依赖

### DeltaForce.Core — 特勤处

**功能说明**

三角洲行动的核心插件，负责玩家匹配、装备管理、交易行和哈夫币系统。作为匹配大厅服务器运行。

**功能特性**

| 功能模块 | 说明 |
|---------|------|
| 小队匹配系统 | 自动将玩家分配到小队（每队最多3人），匹配倒计时提示 |
| 背包数据管理 | 自动保存/恢复玩家背包数据到数据库 |
| 哈夫币系统 | 新玩家初始获得 1000 哈夫币，支持玩家间转账 |
| 交易行系统 | 上架物品、搜索浏览、自动交易、直接出售给系统 |

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/squad` | deltaforce.squad | 加入匹配队列 |
| `/market balance` | deltaforce.market | 查看哈夫币余额 |
| `/market list [页码]` | deltaforce.market | 查看上架物品列表 |
| `/market sell <价格> [数量]` | deltaforce.market | 上架手持物品 |
| `/market sellnpc [数量]` | deltaforce.market | 直接出售给系统 |
| `/market buy <ID>` | deltaforce.market | 购买物品 |
| `/market cancel <ID>` | deltaforce.market | 取消上架物品 |
| `/market my` | deltaforce.market | 查看自己的上架物品 |
| `/market search <关键词>` | deltaforce.market | 搜索物品 |
| `/market pay <玩家> <金额>` | deltaforce.market | 转账哈夫币 |

**配置要点**

- 配置文件位于 `tshock/DeltaCore.json`

```json5
{
  "socket_server": { "address": "0.0.0.0", "port": 7778 },
  "game_server": { "address": "127.0.0.1", "port": 7777 },
  "match_seconds": 60,
  "items": [
    { "name": "金币", "type": 73, "weight": 20, "value": 10000 }
  ]
}
```

**数据库表**

| 表名 | 说明 |
|------|------|
| `deltaforce_inventory` | 玩家背包数据 |
| `deltaforce_currency` | 玩家哈夫币数据 |
| `deltaforce_marketplace` | 交易行物品数据 |

### DeltaForce.Game — 三角洲行动游戏服务器

**功能说明**

三角洲行动的游戏玩法插件，负责实际的游戏逻辑判定，包括死亡掉落、出生点传送、撤离点系统和箱子物品分配。

**功能特性**

| 功能模块 | 说明 |
|---------|------|
| 出生点系统 | 为每个小队分配随机出生点，支持配置多个出生点 |
| 死亡掉落系统 | 玩家死亡时掉落所有装备，变为幽灵状态 |
| 撤离点系统 | 在撤离点范围内停留指定时间后成功撤离 |
| 箱子物品分配 | 游戏开始时重置所有箱子并按权重随机分配物品 |
| 游戏时间管理 | 倒计时提示，时间结束后未撤离玩家丢失装备 |

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/df leave` | deltaforce.game | 主动离开游戏返回特勤处 |
| `/df time` | deltaforce.game | 查看剩余游戏时间 |
| `/df evac` | deltaforce.game | 查看撤离点列表 |

**游戏流程**

1. 特勤处匹配玩家，通过协议通知游戏服务器开始游戏
2. 分配物品到地图中的所有箱子
3. 为每个小队分配出生点并传送玩家
4. 玩家搜刮装备
5. 到达撤离点，停留指定时间后成功撤离
6. 成功撤离的玩家保存装备并返回特勤处
7. 时间结束，未撤离的玩家丢失装备并返回特勤处

**配置要点**

- 配置文件位于 `tshock/DeltaGame.json`

```json5
{
  "socket_server": { "address": "127.0.0.1", "port": 7778 },
  "core_server": { "address": "127.0.0.1", "port": 7777 },
  "match_minute": 10,
  "ready_second": 30,
  "spawn_points": [
    { "name": "北部出生点", "x": 100, "y": 200, "team_id": 1 },
    { "name": "随机出生点", "x": 300, "y": 400 }
  ],
  "spawn_range": 40,
  "evacuation_points": [
    { "name": "主撤离点", "x": 800, "y": 600, "radius": 5, "is_active": true }
  ],
  "evacuation_time_seconds": 10
}
```

### Challenger — 挑战者模式

**功能说明**

挑战者模式是增强游戏难度的综合玩法插件。它修改了 BOSS AI、为装备套装和饰品添加被动技能效果，并实现"血包"机制（玩家受伤时被抽出血包，敌怪可通过血包回血，玩家可拦截血包给自己回血）。

> 此插件在第 12 章（娱乐与小游戏）中已有详细说明，此处仅作简要概述。

**核心机制**

- **血包系统**：玩家受伤时产生血包，敌怪可吸收回血，玩家可拦截血包
- **套装增强**：所有主流套装获得额外被动技能
- **BOSS 魔改**：可独立开关 BOSS AI 修改
- **怪物血量倍率**：可配置所有怪物的血量倍率（仅开启 BOSS 魔改时生效）

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/cenable` | challenger.enable | 启用/解除挑战模式 |
| `/tips` | challenger.tips | 启用/取消内容提示 |
| `/cf` | 无 | 切换或触发装备被动技能 |

### BuildMaster — 建筑大师小游戏

**功能说明**

基于 MiniGamesAPI 框架开发的速建大比拼小游戏。玩法类似于《我的世界》中的建筑大赛。玩家需要在限定时间内完成对应主题的建筑，时间到后由每位玩家进行评分，最后进行排名。

**指令表**

| 指令 | 权限 | 说明 |
|------|------|------|
| `/bm list` | bm.user | 查看房间列表 |
| `/bm join <房间ID>` | bm.user | 加入房间 |
| `/bm leave` | bm.user | 离开房间 |
| `/bm ready` | bm.user | 准备/未准备 |
| `/bm vote <主题>` | bm.user | 投票主题 |
| `/bma list` | bm.admin | 列出所有房间 |
| `/bma create <房间名>` | bm.admin | 创建房间 |
| `/bma remove <房间ID>` | bm.admin | 移除指定房间 |
| `/bma start <房间ID>` | bm.admin | 开启指定房间 |
| `/bma stop <房间ID>` | bm.admin | 关闭指定房间 |
| `/bma smp <房间ID> <人数>` | bm.admin | 设置最大玩家数 |
| `/bma sdp <房间ID> <人数>` | bm.admin | 设置最小玩家数 |
| `/bma swt <房间ID> <秒>` | bm.admin | 设置等待时间 |
| `/bma sgt <房间ID> <秒>` | bm.admin | 设置游戏时间 |
| `/bma sst <房间ID> <秒>` | bm.admin | 设置评分时间 |
| `/bma sp <1/2>` | bm.admin | 选取区域点 |
| `/bma sr <房间ID> <主题>` | bm.admin | 设置游戏区域 |
| `/bma addt <房间ID> <主题名>` | bm.admin | 添加主题 |
| `/bma sh <房间ID> <高度>` | bm.admin | 设置小区域高 |
| `/bma sw <房间ID> <宽度>` | bm.admin | 设置小区域宽 |
| `/bma sg <房间ID> <间隔>` | bm.admin | 设置小区域间隔 |
| `/bma dp <玩家名>` | bm.admin | 设置玩家基础建造背包 |
| `/bma ep` | bm.admin | 设置评分套装 |
| `/bma reload` | bm.admin | 重载配置文件 |

**配置要点**

- 配置文件位于 `tshock/BuildMaster/`
- 需要安装前置插件 MiniGamesAPI
- 需要在旅行模式服务器中运行（使用创造模式）
- 玩家建筑区域要计算好，加起来不要超过总区域宽度
- 玩家建筑区域默认水平创建

**文件结构**

```
tshock/BuildMaster/
├── Config.json       # 主配置（UnlockAll、Range、BanItem）
├── default.json      # 默认建筑套装
├── eva.json          # 评分套装
└── rooms.json        # 房间数据
```
