# 基础库与框架插件

## 分类概述

基础库与框架类插件为其他功能插件提供底层支持，解决以下核心问题：

- **代码复用**：提供通用的指令注册、参数匹配、权限检查等基础设施，避免每个插件重复实现
- **开发效率**：通过声明式 Attribute 简化插件开发流程，降低开发门槛
- **协议通信**：定义标准化的通信协议，支持分布式服务器架构下的数据交换
- **代码生成**：通过源代码生成器在编译期自动生成重复性代码，减少手动编写错误
- **多语言支持**：提供统一的国际化（i18n）基础设施，支持配置文件和服务端消息的本地化

## LazyAPI 插件详解

### 概述

LazyAPI 是整个 TShock 插件生态的基础库，由 cc004 开发。它通过声明式 Attribute 实现插件的快速开发，极大地简化了指令注册、参数匹配、权限校验、Rest API 等常见需求。

- **作者**：cc004（UnrealMultiple 成员）
- **原始出处**：[GitHub - cc004/LazyUtils](https://github.com/cc004/LazyUtils)
- **依赖**：linq2db
- **命令行参数**：`-culture zh|en` 用于切换配置文件语言，默认跟随 TShock 语言

### 架构设计

```
┌─────────────────────────────────────────┐
│           LazyPlugin（抽象基类）            │
│  - AutoLoad() 自动检测 Attribute         │
│  - 自动注册 Command / Config / Rest      │
├─────────────────────────────────────────┤
│              Attribute 体系               │
│  CommandAttribute    指令类声明            │
│  PermissionsAttribute 权限声明             │
│  RealPlayerAttribute  真人玩家校验          │
│  FlexibleAttribute    弹性参数匹配          │
│  AliasAttribute       指令别名             │
│  MainAttribute        主命令标记           │
│  RestAttribute        Rest API 声明        │
│  ConfigAttribute      配置类声明            │
├─────────────────────────────────────────┤
│              参数自动转换系统               │
│  int / long / TSPlayer / bool / string   │
│  UserAccount / DateTime                  │
└─────────────────────────────────────────┘
```

### Attribute 详解

#### CommandAttribute（类级别）

标记一个类为指令处理器，LazyAPI 会在插件实例化时自动检测并注册。

```csharp
[Command("test")]
public class TestCommand
{
    public static void Run(CommandArgs args)
    {
        // 方法内部实现
    }
}
```

#### 子命令系统

默认情况下，类中的每个静态公开方法都会成为以方法名命名的子命令：

```csharp
[Command("test")]
public class TestCommand
{
    // 子命令: /test run
    public static void Run(CommandArgs args) { }

    // 通过 AliasAttribute 添加别名: /test go
    [Alias("go")]
    public static void Execute(CommandArgs args) { }
}
```

#### MainAttribute

标记方法为根命令，使 `/test` 直接执行该方法而无需子命令：

```csharp
[Command("test")]
public class TestCommand
{
    [Main]
    public static void Execute(CommandArgs args) { }
}
```

#### PermissionsAttribute

为指令附加权限节点，自动进行权限校验：

```csharp
[Command("test")]
[Permissions("test")]          // 类级别权限
public class TestCommand
{
    [Permissions("test.use")]  // 方法级别权限（更细粒度）
    public static void Execute(CommandArgs args) { }
}
```

#### RealPlayerAttribute

阻止机器人或非玩家用户执行指令，LazyAPI 自动判断并阻止：

```csharp
[Command("test")]
public class TestCommand
{
    [RealPlayer]
    public static void Execute(CommandArgs args) { }
}
```

#### FlexibleAttribute

用于处理参数数量不固定的场景（如分页），放宽参数匹配限制：

```csharp
[Command("test")]
public class TestCommand
{
    [Flexible]
    public static void List(CommandArgs args)
    {
        var list = new List<string> { /* ... */ };
        args.SendPage(list, 1, new PaginationTools.Settings
        {
            MaxLinesPerPage = 30,
            HeaderFormat = "列表 ({0}/{1})："
        });
    }
}
```

### 参数自动转换

LazyAPI 支持在方法签名中直接声明参数类型，自动从 `args.Parameters` 中解析转换：

```csharp
[Command("test")]
public class TestCommand
{
    public static void Execute(CommandArgs args, string name, int age, TSPlayer target)
    {
        // name 自动匹配第一个参数
        // age 自动转换为 int
        // target 自动匹配在线玩家
    }
}
```

**支持的类型**：

| 类型 | 说明 |
|------|------|
| `int` | 整数 |
| `long` | 长整数 |
| `TSPlayer` | 在线玩家对象 |
| `bool` | 布尔值 |
| `string` | 字符串 |
| `UserAccount` | TShock 用户账户 |
| `DateTime` | 日期时间 |

参数默认严格匹配——数量和类型必须完全一致，否则不执行。

### Rest API 支持

通过 `RestAttribute` 和 `RestPathAttribute` 自动注册 REST API 路由：

```csharp
[Rest("/api/test")]
public class TestRest
{
    public static object Execute(RestRequestArgs args)
    {
        return new { message = "Hello" };
    }
}
```

### 配置系统

通过 `ConfigAttribute` 声明配置类，支持自动生成默认配置和多语言配置：

```csharp
[Config("MyPlugin")]
public class MyConfig
{
    public string Option1 { get; set; } = "default";
}
```

## Economics.Core 插件详解

### 概述

Economics.Core 是 Economics 系列经济插件的核心前置，提供多货币系统和服务层 API。它是延续 POBC 设计思路的新经济系统，本身不提供游戏功能，只提供货币系统和对外 API。

- **作者**：少司命（千亦 修复 bug）
- **版本**：v3.0.0.0（重大架构重构）
- **依赖**：无（其他 Economics 子插件均依赖此插件）
- **配置文件位置**：`tshock/Economics/Economics.json`

## DeltaForce.Protocol 插件详解

### 概述

三角洲行动通信协议库，定义特勤处服务器（Core）和游戏服务器（Game）之间的通信数据包格式和处理逻辑。

- **作者**：少司命
- **依赖**：无（.NET 9.0 独立库）
- **用途**：三角洲行动玩法的底层通信基础设施

### 协议类型

#### 数据包类型

```csharp
public enum PacketType : byte
{
    Request, Response,           // 系统
    Connect, Disconnect,         // 连接
    GameState, GameStateResponse, // 游戏状态
    Heartbeat, HeartbeatResponse, // 心跳
    PlayerInventoryRequest/Response, // 背包数据
    SquadDataRequest/Response,   // 小队数据
    ItemListRequest/Response,    // 物品清单
    ClientIdentity/Response,     // 客户端标识
    PlayerPositionRequest/Response, // 玩家位置
}
```

#### 主要数据包

| 数据包 | 说明 |
|--------|------|
| GameStatePacket | 通知游戏开始/结束，包含游戏模式和时间 |
| ItemListPacket | 同步物品配置列表（名称、类型、权重、价值） |
| SquadDataPacket | 同步小队数据和成员信息 |
| PlayerInventoryPacket | 同步玩家背包数据 |
| SaveInventoryPacket | 保存玩家背包到特勤处 |
| HeartbeatPacket | 维持连接和检测断开 |
| ClientIdentityPacket | 身份验证 |

### 序列化

二进制序列化，支持基础类型、字符串、数组、嵌套对象。使用 `[PacketType(PacketType.XXX)]` 特性标记。

### 处理器模式

```csharp
// 请求处理器基类
public abstract class RequestHandlerBase<TRequest, TResponse>
    where TRequest : IRequestPacket
    where TResponse : IResponsePacket, new()
{
    public abstract TResponse Handle(TRequest request);
}

// 处理器注册
processor.RegisterHandler<ItemListHandler>();
processor.RegisterHandlersFromAssembly(Assembly.GetExecutingAssembly());
```

### 使用示例

```csharp
// 发送请求
var request = new ItemListRequestPacket();
var response = await client.RequestAsync<ItemListRequestPacket, ItemListResponsePacket>(request);
```

## Shared 共享库

### 概述

Shared 是一个轻量级共享库，主要提供国际化（i18n）基础工具。该库通过 `Localizer.I18n` 命名空间提供全局可用的 `GetString` 方法。

- **文件**：`I18n.cs`
- **功能**：基于 GetText 的 .mo 文件翻译系统
- **用途**：为插件提供多语言字符串翻译支持

### 核心功能

```csharp
// 全局 using: static Localizer.I18n
GetString("Hello World");                    // 翻译普通字符串
GetString("Hello {0}", name);                // 翻译带参数的字符串
GetPluralString("item", "items", count);     // 复数形式翻译
GetParticularString("context", "text");      // 上下文特定翻译
```

### 工作机制

1. 从 TShock 的 `TranslationCultureInfo` 获取当前语言设置
2. 加载对应语言的 `.mo` 翻译文件
3. 若找不到翻译文件则回退为原文

## SourceGen 源代码生成器

### 概述

SourceGen 是一个 C# 源代码生成器（Incremental Generator），在编译期自动分析 `ProgressType` 枚举并生成进度匹配代码。

- **文件**：`ProgressSourceGenerator.cs`
- **框架**：Microsoft CodeAnalysis（Roslyn）
- **用途**：自动生成 `IProgressMap` 实现类，用于游戏进度检测

### 生成机制

```
编译时检测 ProgressType 枚举
        ↓
解析每个枚举成员的 Attribute 元数据
        ↓
为每个枚举值生成 IProgressMap 实现类
        ↓
生成 ProgressMatchAttribute 用于运行时匹配
```

### 生成的代码示例

```csharp
[ProgressMatch(ProgressType.Skeletron, "骷髅王")]
internal class SkeletronMap : IProgressMap
{
    public bool GetStatus(Player? ply = null)
    {
        return NPC.downedBoss3 == true;  // 骷髅王已被击杀
    }
}
```

### 自定义 Attribute

| Attribute | 用途 |
|-----------|------|
| `ProgressNameAttribute` | 定义进度别名（支持多语言名称匹配） |
| `ProgressMapIDAttribute` | 关联 NPC/图格 ID |
| `ProgressMapAttribute` | 指定映射到游戏内字段 |
